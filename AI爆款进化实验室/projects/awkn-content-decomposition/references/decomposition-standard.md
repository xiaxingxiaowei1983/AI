# 内容拆解详细标准

## 核心原则
内容拆解的目标是：**从"读懂"到"能用"**。不仅要理解内容，更要提炼出可执行的方法论。

## 拆解的三个层次

### 第一层：核心框架（必填）
**目的**：快速把握内容的本质和价值

包含要素：
1. **核心命题**：1句话概括核心观点
   - 示例："认知边界决定人生边界"
   
2. **核心观点**：提炼3-5个核心观点
   - 每个观点1-2句话
   - 示例："人类三重大脑模型揭示了成长的底层困境"
   
3. **核心价值**：说明价值所在
   - 解决了什么问题？
   - 提供了什么方法？
   - 示例："提供了从平庸到卓越的实践路径"
   
4. **适用场景**：分析适合的人群和使用场景
   - 谁需要这个内容？
   - 在什么情况下使用？
   - 示例："职场新人、工作多年陷入瓶颈"

---

### 第二层：详细内容拆解（必填，必须展开）

**目的**：深度理解内容，提炼方法论

#### 如果是一本书：

**每个章节必须包含以下5个部分**：

1. **底层逻辑**（必须详细）
   - 揭示的核心原理、科学依据
   - 为什么这个问题会存在？
   - 背后的根本原因是什么？
   - 示例："人类大脑经过数百万年的进化，形成了三层结构"
   
2. **核心概念**（必须定义清晰）
   - 关键概念的定义和解释
   - 用简单易懂的语言解释
   - 示例："元认知是对'思考过程'的认知"
   
3. **方法论**（必须具体）
   - 具体的步骤、框架、模型
   - 如何操作？
   - 有哪些方法可以使用？
   - 示例："第一步：觉察自己的行为模式；第二步：理解冲动是正常的"
   
4. **实践建议**（必须可执行）
   - 可执行的行动建议
   - 每天/每周应该做什么？
   - 如何应用到实际生活中？
   - 示例："每天进行5分钟元认知反思：今天我做了哪些决策？"
   
5. **案例说明**（必须具体）
   - 具体的案例或例子
   - 用实际场景说明
   - 示例："为什么我们明知道要学习，却刷手机停不下来？本能脑追求即时满足"

#### 如果是一篇文章：

必须包含以下5个部分：

1. **中心思想**：文章的核心论点
2. **论证逻辑**：文章的论证过程
3. **关键论据**：支撑论点的关键论据
4. **实践建议**：给出的具体建议
5. **总结**：文章的结论

---

### 第三层：总结升华（必填）

**目的**：提炼核心启示，规划实践路径

包含要素：
1. **核心启示**（3-5个关键点）
   - 这本书/文章的核心思想是什么？
   - 对读者的最大价值是什么？
   
2. **实践路线图**（可选）
   - 第1-2周：做什么
   - 第3-4周：做什么
   - 第2-3个月：做什么
   - 第4-6个月：做什么
   
3. **最终目标**（可选）
   - 通过学习这个内容，期望达到什么目标？

---

## 拆解质量检查清单

完成拆解后，逐项检查：

### 结构完整性
- [ ] 是否包含核心框架（核心命题、核心观点、核心价值、适用场景）
- [ ] 是否包含详细内容拆解（每个章节都有底层逻辑、核心概念、方法论、实践建议、案例说明）
- [ ] 是否包含总结升华

### 内容质量
- [ ] 核心命题是否清晰（1句话概括）
- [ ] 核心观点是否提炼到位（3-5个，每个1-2句话）
- [ ] 底层逻辑是否详细展开（不能只说"要耐心"，要解释"为什么要耐心"）
- [ ] 核心概念是否定义清晰（用简单易懂的语言）
- [ ] 方法论是否具体（有步骤、有框架、有模型）
- [ ] 实践建议是否可执行（不能只是理论，要有具体的行动）
- [ ] 案例说明是否具体（有实际场景，可参考）

### 常见错误检查
- [ ] 是否只提炼了核心观点，缺少详细展开？
- [ ] 是否只列举了概念，缺少解释和案例？
- [ ] 是否只有理论，缺少实践建议？
- [ ] 是否只有抽象描述，缺少具体例子？

## 常见错误及纠正

### 错误1：只提炼核心观点，缺少详细展开

**错误示例**：
```
核心观点：
1. 要有耐心
2. 要专注
3. 要反思
```

**纠正后**：
```
底层逻辑：
人类大脑经过数百万年的进化，形成了三层结构：本能脑、情绪脑、理智脑。理智脑虽然高级，但力量远不及本能脑和情绪脑，所以我们很难保持耐心。

获得耐心的三个路径：
1. 坦然接纳——对急躁不自责，温和对话
2. 延迟满足——先做重要事后娱乐
3. 赋予意义——看清困难之事的价值

实践建议：
当感到急躁时，告诉自己"这是正常的"，然后深呼吸3次，给理智脑争取时间。
```

---

### 错误2：只列举概念，缺少解释和案例

**错误示例**：
```
核心概念：
- 元认知
- 专注力
- 学习力
```

**纠正后**：
```
核心概念：
元认知是对自身"思考过程"进行认知和理解的能力，就像是你有一个"观察者"在监控你的思考。

案例说明：
普通人的思考："我觉得应该换工作，因为工资高一点。"（被第一反应控制）

高手的思考（元认知）："等等，我现在在做这个决策。让我监控一下我的思考过程：1. 我只考虑了工资，忽略了其他因素；2. 我是不是被高薪诱惑了？3. 让我重新审视：换工作的利弊各是什么？"

通过元认知监控，避免被表面信息误导，做出更理性的决策。
```

---

### 错误3：只有理论，缺少实践建议

**错误示例**：
```
方法论：
- 要有最小行动单元
- 要建立正向反馈
```

**纠正后**：
```
方法论：
最小行动单元是将大目标拆解为极小、无法拒绝的行动。

实践建议：
将大目标拆解为最小行动单元：
- 大目标：写一本书
- 拆解：写一章 → 写一节 → 写一段 → 写一句话 → 打开文档
- 最小行动：打开文档，写一句话

使用5分钟法则：
- 不想做某事时，告诉自己"只做5分钟"
- 5分钟后不想做就放弃，想继续就继续
- 大多数时候你会继续做

案例说明：
如何通过最小行动单元养成晨跑习惯？
- 第1周：每天早上起床后，穿运动鞋（2分钟）
- 第2周：穿上运动鞋后，出门走5分钟（5分钟）
- 第3周：走5分钟后，慢跑1公里（15分钟）
- 第4周：逐步增加到3公里
- 第2个月：稳定在5公里
```

## 拆解示例结构

完整的拆解应该像这样：

```
【内容拆解报告】

# 一、核心框架
- 核心命题：[1句话概括核心观点]
- 核心观点：
  1. [观点1]
  2. [观点2]
  3. [观点3]
- 核心价值：[价值所在]
- 适用场景：[适用人群和场景]

# 二、详细内容拆解

## 第1章：[章节标题]
### 底层逻辑
[揭示的核心原理、科学依据，详细展开]

### 核心概念
[关键概念的定义和解释，用简单易懂的语言]

### 方法论
[具体的步骤、框架、模型，可操作]

### 实践建议
[可执行的行动建议，具体到每天/每周]

### 案例说明
[具体的案例或例子，有实际场景]

[继续其他章节...]

# 三、总结升华
核心启示：
1. [启示1]
2. [启示2]
3. [启示3]

实践路线图：
- 第1-2周：[做什么]
- 第3-4周：[做什么]
- 第2-3个月：[做什么]
- 第4-6个月：[做什么]

最终目标：
[期望达到的目标]
```

## 提示词使用指南

当你需要拆解内容时，可以这样使用：

**拆解请求**：
```
请按照"内容拆解详细标准"帮我拆解这本书/文章：
[书名/文章链接/原始内容]

要求：
1. 详细展开，不能只有简短概括
2. 每个章节都要包含底层逻辑、核心概念、方法论、实践建议、案例说明
3. 实践建议必须可执行
```

**质量检查请求**：
```
请按照"拆解质量检查清单"检查我的拆解是否符合标准：
[你的拆解内容]

如果有不符合的地方，请指出并给出改进建议。
```
