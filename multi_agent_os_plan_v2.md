# 多角色 AI 协作操作系统 - 精准实施计划

## 核心理念

**主线任务**：利用 OpenClaw 原生能力完成配置
**支线任务**：开发缺失的适配器和增强功能

---

## 第一阶段：主线基建（利用原生能力）

### [ ] 任务 1.1: 配置 Discord 通道
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 在全局配置中添加 Discord 通道
  - 配置 Bot Token 和权限
  - 指定服务器 ID 和频道
- **成功标准**:
  - Discord 消息能被 OpenClaw 接收
  - 系统能向 Discord 发送消息
- **测试要求**:
  - `programmatic` TR-1.1.1: Discord 通道配置正确
  - `programmatic` TR-1.1.2: 消息接收成功

### [ ] 任务 1.2: 配置多角色路由与隔离
- **优先级**: P0
- **依赖**: 任务 1.1
- **描述**:
  - 为每个智能体分配独立工作区（已有）
  - 配置路由绑定规则（来源 → 角色映射）
  - 配置独立的状态目录
- **成功标准**:
  - 不同来源消息路由到正确角色
  - 智能体物理隔离生效
- **测试要求**:
  - `programmatic` TR-1.2.1: 路由规则配置正确
  - `programmatic` TR-1.2.2: 消息路由到正确智能体

### [ ] 任务 1.3: 配置智能体间通信权限
- **优先级**: P0
- **依赖**: 任务 1.2
- **描述**:
  - 在全局配置中添加 agentCommunication
  - 配置 canCommand 权限列表
  - 配置消息转发规则
- **成功标准**:
  - 大宗师能派发任务给大客栈
  - 权限边界清晰
- **测试要求**:
  - `programmatic` TR-1.3.1: 通信权限配置正确
  - `programmatic` TR-1.3.2: 智能体间通信正常

### [ ] 任务 1.4: 配置沙箱管控与权限矩阵
- **优先级**: P1
- **依赖**: 任务 1.3
- **描述**:

- **测试要求**:

## 配置示例

### Discord 通道配置
```json
{
  "channels": {
    "discord": {
      "enabled": true,
      "botToken": "${DISCORD_BOT_TOKEN}",
      "serverId": "${DISCORD_SERVER_ID}",
      "channels": {
        "运营": "1234567890",
        "技术": "0987654321"
      }
    }
  }
}
```

### 智能体路由配置
```json
{
  "agents": {
    "routing": {
      "feishu:核心管理群": "main",
      "discord:运营频道": "coo",
      "discord:技术频道": "business"
    }
  }
}
```

### 智能体间通信配置
```json
{
  "agentCommunication": {
    "enabled": true,
    "permissions": {
      "main": {
        "canCommand": ["coo", "business", "life"],
        "canForwardTo": ["feishu:核心管理群"]
      },
      "coo": {
        "canCommand": ["business"],
        "canForwardTo": ["discord:运营频道"]
      }
    }
  }
}
```

### 沙箱权限配置
```json
{
  "agents": {
    "list": [
      {
        "id": "business",
        "sandbox": {
          "enabled": true,
          "executionLevel": "isolated",
          "allowedTools": ["read_file", "write_file", "execute_command"],
          "blockedTools": ["system_config", "modify_gateway"]
        }
      }
    ]
  }
}
```

## 成功标准

1. **双通道运行**：Discord 和飞书双通道都能正常工作
2. **角色隔离**：智能体之间物理隔离，互不干扰
3. **权限明确**：每个角色的权限边界清晰
4. **协作顺畅**：智能体之间能够正常通信和协作
5. **稳定运行**：系统能够长期稳定运行，资源占用可控
