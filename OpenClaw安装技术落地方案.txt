步骤1：安装Node.js

- 推荐版本：v22.x LTS（长期支持版）

2. 勾选 "Automatically install the necessary tools"（自动安装必要工具）
3. 点击 "Next" 完成安装
  
1.3 验证安装

打开PowerShell，执行：
```PowerShell
node --version
npm --version```

验收标准：
- Node.js版本 ≥ 22.0.0
- npm版本 ≥ 10.0.0

步骤2：安装OpenClaw

2.1 执行安装命令

```PowerShell
npm install -g openclaw@latest --ignore-scripts```

⚠️ 重要：Windows系统必须添加 --ignore-scripts 参数，否则会因为原生依赖编译失败而报错。

2.2 验证安装

```PowerShell
openclaw --version```

验收标准：
- 显示版本号（如：2026.2.22-2）
- 无报错信息
  

---

步骤3：配置模型

3.1 获取API Key

火山引擎/豆包API Key获取步骤：
1. 访问火山引擎控制台：https://console.volcengine.com/
2. 登录/注册账号
3. 进入"方舟"（ARK）服务
4. 创建推理接入点，获取：
  - API Key（以9330开头），我的是豆包1.8模型， 所以是这个开头，现在豆包已经更新到2.0版，所以开头不一定是9330
  - Endpoint ID（以ep-开头）
    
3.2 配置模型

```PowerShell
# 设置模型provider
openclaw config set models.providers.volcano.baseUrl "https://ark.cn-beijing.volces.com/api/v3"

# 设置API Key（替换为你的Key）
openclaw config set models.providers.volcano.apiKey "你的API_KEY"

# 设置模型ID（替换为你的Endpoint ID）
openclaw config set models.providers.volcano.models "[{\"id\":\"你的ENDPOINT_ID\",\"name\":\"Doubao Pro\"}]"

# 设置默认模型
openclaw config set agents.defaults.model.primary "volcano/你的ENDPOINT_ID"```

3.3 验证配置

```PowerShell
openclaw models list```

验收标准：
- 显示 volcano/ep-xxx 模型
- 无 "missing" 标签

2.2 详细复盘


---

坑点1：Windows原生依赖编译失败

问题描述：
```Plain Text
gyp ERR! find Python
gyp ERR! configure error
gyp ERR! stack Error: Can't find Python executable "python"```

根因分析：
- OpenClaw包含原生Node模块
- Windows缺少编译工具（Python、Visual Studio Build Tools）
- npm默认会尝试编译原生模块
  
解决方案：
```PowerShell
npm install -g openclaw@latest --ignore-scripts```

预防措施：
- 安装时必须添加 --ignore-scripts 参数
- 或预先安装 Windows Build Tools
  

---

坑点2：默认模型漂移（Unknown model）

问题描述：
```Plain Text
Unknown model: anthropic/ep-20260108213536-xkxvw
Unknown model: volcengine-plan/ark-code-latest```

根因分析：
- 配置文件中残留了Anthropic provider
- 默认模型没有指定provider前缀
- 存在错误的volcengine-plan provider
  
解决方案：
```PowerShell
# 1. 查看当前配置
openclaw config get

# 2. 删除错误的provider
openclaw config unset models.providers.anthropic
openclaw config unset models.providers.volcengine
openclaw config unset models.providers.volcengine-plan

# 3. 设置正确的默认模型（必须带provider前缀）
openclaw config set agents.defaults.model.primary "volcano/你的ENDPOINT_ID"```

预防措施：
- 默认模型必须包含provider前缀（如 volcano/ep-xxx）
- 安装后立即检查配置文件
- 删除不需要的provider
  

---

坑点3：错误的provider配置

问题描述：
```JSON
{
  "volcengine": {
    "apiKey": "VOLCANO_ENGINE_API_KEY"  // 占位符，不是真实Key
  }
}```

根因分析：
- 某些provider配置使用了占位符
- 实际使用的是 volcano provider，但配置了 volcengine
  
解决方案：
删除使用占位符的provider配置。

预防措施：
- 只保留有真实API Key的provider
- 检查配置文件中的apiKey字段
  

---

坑点4：Dashboard "Not Found"

问题描述：
访问 http://127.0.0.1:18789/ 或 /chat 返回 "Not Found"

根因分析：
- 可能是UI资源未正确部署
- 或沙箱环境限制
  
解决方案：
使用CLI命令替代Dashboard测试：
```PowerShell
openclaw agent --message "你好"```

预防措施：
- 不依赖Dashboard进行测试
- Dashboard问题不影响核心功能
  

---

坑点5：Canvas "Missing file"

问题描述：
访问Canvas页面显示：
```Plain Text
Missing file. Create C:\Users\用户名\.openclaw\canvas/index.html```

根因分析：
- Canvas目录或文件不存在
  
解决方案：
```PowerShell
# 创建canvas目录
New-Item -ItemType Directory -Path "$env:USERPROFILE\.openclaw\canvas" -Force

# 创建index.html文件
Set-Content -Path "$env:USERPROFILE\.openclaw\canvas\index.html" -Value "<html><body><h1>Canvas</h1></body></html>"```

预防措施：
- Canvas功能非核心，可忽略
- 或手动创建文件
  

---

坑点6：沙箱文件访问限制

问题描述：
```Plain Text
TRAE Sandbox Error: hit restricted
Not allow operate files: "C:\\Users\\L\\.openclaw\\agents\\main\\agent\\auth.json.lock"
EPERM: operation not permitted```

根因分析：
- Trae IDE的沙箱环境限制了部分文件访问
- 某些操作需要管理员权限
  
解决方案：
- 在Trae IDE外部运行PowerShell
- 或以管理员身份运行
  
预防措施：
- 重要操作在独立终端执行
- 避免在沙箱环境中执行需要文件锁的操作

第四部分：常见问题FAQ

Q1：安装时出现node-gyp错误怎么办？

A：Windows系统必须添加 --ignore-scripts 参数：
```PowerShell
npm install -g openclaw@latest --ignore-scripts```


---

Q2：出现 "Unknown model" 错误怎么办？

A：检查默认模型配置，确保包含provider前缀：
```PowerShell
# 查看当前配置
openclaw config get agents.defaults.model.primary

# 应该显示：volcano/ep-xxx
# 如果只显示 ep-xxx，需要修复：
openclaw config set agents.defaults.model.primary "volcano/YOUR_ENDPOINT_ID"```


---

Q3：Dashboard显示 "Not Found" 怎么办？

A：这是已知问题，不影响核心功能。使用CLI命令测试：
```PowerShell
openclaw agent --message "你好"```


---

Q4：如何查看配置文件？

A：配置文件位置：
```Plain Text
Windows: C:\Users\用户名\.openclaw\openclaw.json
macOS/Linux: ~/.openclaw/openclaw.json```

查看命令：
```PowerShell
openclaw config get```


---

Q5：如何重置配置？

A：删除配置文件后重新配置：
```PowerShell
# 备份
Copy-Item "$env:USERPROFILE\.openclaw\openclaw.json" "$env:USERPROFILE\.openclaw\openclaw.json.backup"

# 删除
Remove-Item "$env:USERPROFILE\.openclaw\openclaw.json"

# 重新运行配置向导
openclaw configure```

🌈 五、最核心的四类 Key（通俗解释）

---
✅ 1. API Key（最常用）
👉 这是调用 AI 的通行证
格式类似：
```Plain Text

- 最常用
- 最重要

✅ 2. Endpoint（模型编号）
👉 告诉系统你要用哪个模型
格式类似：
```Plain Text

你可以理解为：
👉 API Key 是身份证
👉 Endpoint 是医生号
没有 Endpoint：

```Plain Text

```Plain Text

👉 火山云平台的全局身份

还有一个，就是app_id，这个是什么鬼，和前面那一堆又是什么关系
这个问题非常好，而且很多人都会在这里卡住。
 我给你一次讲透，让你以后看到任何云厂商的 app_id / project_id / tenant_id 都不会再懵 🌿。

---
🌈 一、先给你一句话答案
👉 app_id 不是 Key，也不是用来调用模型的。
 👉 它是“项目或应用的身份编号”。
它的作用是：
 👉 标识是谁在调用、哪个应用在调用。

---
🌿 二、用一个最容易懂的比喻
还是用我们刚刚的医院模型。
暂时无法在飞书文档外展示此内容
也就是说：
👉 API Key 是你的身份证
 👉 Endpoint 是医生
 👉 app_id 是你的病例号

---
🌈 三、为什么需要 app_id？
因为在企业级系统里，很多公司：
- 一个公司有多个产品
- 一个产品有多个环境
- 多个团队共享一个云账号
所以平台需要知道：
👉 是谁在用这个 API。
而不是只看 API Key。

---
举个真实场景
比如你未来会有：
- 木木简历顾问
- AI 分身
- 小脑瓜
- 书瑶实验室
如果都用同一个火山账号：
👉 就需要不同 app_id 来区分。
否则：
- 账单混乱
- 权限无法隔离
- 数据统计困难

---
🌿 四、它和前面那些 Key 的关系
这是核心逻辑：
火山账号
   ↓
API Key（身份认证）
   ↓
app_id（区分具体应用）
   ↓
Endpoint（调用具体模型）
也就是说：
👉 app_id 是在 API Key 之上的一层。

---

因为很多工具帮你隐藏了。
例如：
- OpenAI

- DeepSeek

👉 自动在后台生成。
但火山更偏企业级：
👉 有些 SDK 或系统会要求你填。

通常在以下场景：
