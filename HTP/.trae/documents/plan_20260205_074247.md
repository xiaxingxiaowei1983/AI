# 修改generateIllustration函数切换到火山文生图API

## 问题分析
当前项目中的generateIllustration函数使用的是阿里的dashscope.aliyuncs.com接口，需要切换到火山文生图的代理地址/api/ark-image，以解决插画跨域问题并适配火山参数格式。

## 解决方案
替换htpAnalysisService.ts中的generateIllustration函数，使用用户提供的最终版插画生成函数，适配火山文生图的参数格式，使用Vite代理的同源地址，解决跨域问题。

### 修改内容

1. **更新函数签名**：从接受对象参数改为直接接受prompt字符串
2. **适配火山参数**：使用火山文生图的参数格式
3. **使用代理地址**：请求Vite代理的同源地址 /api/ark-image，无跨域
4. **更新响应处理**：适配火山文生图的返回格式

### 修改后的代码

```typescript
// 生成插画函数
export async function generateIllustration(prompt: string): Promise<string> {
  try {
    console.log("🎨 生成line_optimized类型插画...");
    console.log("插画提示词:", prompt);
    console.log("📡 发送火山文生图API请求（Vite代理，无跨域）...");

    // 火山文生图请求参数（适配火山规范，和你的提示词/比例完全匹配）
    const requestData = {
      model: ARK_IMAGE_MODEL, // 火山文生图接入点ID
      prompt: prompt, // 你的治愈系提示词，不变
      n: 1, // 生成1张图
      size: ARK_IMAGE_ASPECT, // 插画比例 223:100
      style: "水彩", // 贴合你的提示词风格，可选
      quality: "high" // 高质量生成
    };

    // 🌟 请求Vite代理的同源地址 /api/ark-image，无跨域
    const response = await fetch(ARK_IMAGE_URL, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${ARK_API_KEY}` // 通用火山API Key，无需额外配置
      },
      body: JSON.stringify(requestData)
    });

    // 处理响应（火山文生图返回格式和OpenAI兼容，易解析）
    if (!response.ok) {
      const err = await response.json().catch(() => ({ message: response.statusText }));
      throw new Error(`火山文生图失败：${response.status} - ${err.message}`);
    }

    const result = await response.json();
    console.log("✅ 火山文生图成功，返回结果:", result);

    // 提取火山返回的图片URL（核心：火山格式为 result.data[0].url）
    const imageUrl = result.data?.[0]?.url;
    if (!imageUrl) throw new Error("火山未返回有效插画地址");

    return imageUrl; // 返回图片地址供前端渲染
  } catch (error: any) {
    console.error("❌ 插画生成过程中发生错误:", error.message || error);
    throw error; // 抛出错误让上层捕获
  }
}
```
