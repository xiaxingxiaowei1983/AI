# 修复HTP分析结果页面的路由传参问题

## 问题分析

通过分析用户提供的信息，我发现了以下关键问题：

1. **路由跳转问题**：
   - 前端使用路由跳转（`Maps('/result')`）从加载页跳转到结果页
   - 在React Router中，组件之间的Props不会自动跨页面传递

2. **数据传递问题**：
   - `ResultPage`组件期望通过props接收`analysisResult`数据
   - 但路由跳转后，它是一个独立的页面，没有父组件给它传参
   - 导致`ResultPage`组件接收到的数据为空，页面显示空白

3. **核心症状**：
   - 日志显示：`AppContext.tsx:73 风险等级为 low 跳转到 result 页面`
   - 但`ResultPage`组件无法接收到分析结果数据

## 解决方案

修改`ResultPage`组件，使其能够从路由状态（Location State）中抓取数据：

1. **修改 `src/sections/ResultPage.tsx`**：
   - 导入`useLocation`和`useNavigate`钩子
   - 实现双重数据源抓取：
     - 先检查Props是否传递了数据（组件直接调用模式）
     - 再检查路由状态是否传递了数据（navigate跳转模式）
   - 添加详细的调试日志，方便查看数据流转过程

2. **检查跳转代码**：
   - 确保跳转代码包含state数据，例如：
     ```typescript
     navigate('/result', {
       state: {
         result: analysisResult // 把数据塞进 state 里带过去
       }
     });
     ```

## 实施步骤

1. **修改前端代码**：
   - 覆盖 `src/sections/ResultPage.tsx` 文件，实现从路由状态中抓取数据
   - 添加详细的调试日志，方便查看数据流转过程

2. **检查跳转代码**：
   - 检查 `AppContext.tsx` 或 `LoadingPage.tsx` 中的跳转逻辑
   - 确保跳转代码包含state数据

3. **测试完整流程**：
   - 重新运行分析流程
   - 查看浏览器控制台，确认数据是否正确传递
   - 验证 `ResultPage` 组件是否能够正确显示分析结果

## 预期结果

通过上述修改，`ResultPage`组件将能够：
1. 从路由状态中抓取分析结果数据
2. 智能提取 `clientInsightReport` 字段并渲染成Markdown格式
3. 显示带有正确格式的分析结果
4. 在数据丢失时提供友好的错误提示和返回按钮

这样可以确保用户在完成分析后，能够正确看到分析结果，而不是面对一个空白页面。
