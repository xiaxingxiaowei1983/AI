# 添加Vite Proxy配置解决跨域问题

## 问题分析
当前项目在调用火山豆包API时遇到了跨域问题，这是前端开发中常见的问题。根据用户的要求，需要通过配置Vite Proxy来解决所有跨域问题，包括火山对话API和文生图API。

## 解决方案
修改vite.config.ts文件，添加代理规则，将前端对火山API的请求通过本地代理转发，从而解决跨域问题。

### 修改vite.config.ts文件

1. **添加envPrefix配置**：保留之前的配置，适配.env的VITE_前缀变量
2. **添加server配置**：设置端口、自动打开浏览器
3. **添加proxy配置**：
   - 代理火山对话API：前端请求 /api/ark-chat → 转发到火山对话接口
   - 代理火山文生图API：前端请求 /api/ark-image → 转发到火山文生图接口
4. **保留原有的插件配置**：保持inspectAttr()和react()插件
5. **保留原有的resolve配置**：保持@别名配置

### 修改后的vite.config.ts文件

```typescript
import path from "path"
import react from "@vitejs/plugin-react"
import { defineConfig } from "vite"
import { inspectAttr } from 'kimi-plugin-inspect-react'

// https://vite.dev/config/
export default defineConfig({
  base: './',
  envPrefix: 'VITE_', // 保留之前的配置，适配.env的VITE_前缀变量
  server: {
    port: 5173,
    open: true,
    // 核心：统一代理火山对话+文生图API，解决所有跨域
    proxy: {
      // 代理火山【对话API】：前端请求 /api/ark-chat → 转发到火山对话接口
      '/api/ark-chat': {
        target: 'https://ark.cn-beijing.volces.com/api/v3/chat/completions',
        changeOrigin: true, // 必须开！模拟跨域Origin，解决CORS核心
        secure: true, // 火山是HTTPS，设为true
        rewrite: (path) => path.replace(/^\/api\/ark-chat/, '') // 重写路径，去掉代理前缀
      },
      // 代理火山【文生图API】：前端请求 /api/ark-image → 转发到火山文生图接口
      '/api/ark-image': {
        target: 'https://ark.cn-beijing.volces.com/api/v3/images/generations',
        changeOrigin: true,
        secure: true,
        rewrite: (path) => path.replace(/^\/api\/ark-image/, '')
      }
    }
  },
  plugins: [inspectAttr(), react()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

## 验证步骤

1. **重启开发服务**：停止当前运行的服务，重新执行`npm run dev`
2. **测试API调用**：尝试上传一张房树人绘画作品进行分析
3. **检查浏览器控制台**：确认没有跨域错误
4. **验证API响应**：确认能够成功获取火山豆包API的响应

## 预期结果
- 开发服务器运行在http://localhost:5173/
- 浏览器不再显示跨域错误
- 能够成功调用火山豆包API进行房树人分析
- 能够成功调用火山文生图API生成插画

## 核心原理
- 前端不再直调火山的外网地址，而是请求同源的本地地址（/api/ark-chat、/api/ark-image）
- 浏览器认为是同域请求，不会触发CORS拦截
- Vite作为服务端，后台把请求转发到火山的真实接口（服务端请求无跨域限制）
- Vite自动添加CORS响应头，前端正常接收结果

这个配置方案一次解决所有跨域问题，包括火山对话API和文生图API，是最有效的解决方案。
