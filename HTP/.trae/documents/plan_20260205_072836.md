# 修改analyzeDrawingWithAI函数处理图片数据

## 问题分析
当前项目中，用户在CanvasPage.tsx页面绘制的图片会被转换为Base64编码的字符串，然后传递给analyzeDrawingWithAI函数进行分析。但是，analyzeDrawingWithAI函数在构建请求体时，只传递了文本内容，没有包含图片数据，导致调用智能体时出现"InvalidParameter: 请求缺少必要参数或参数无效，请检查后重试"的错误。

## 解决方案
修改htpAnalysisService.ts文件中的analyzeDrawingWithAI函数，使其能够正确处理Base64编码的图片，并将其包含在请求体中，按照Doubao-1.5-vision-pro多模态模型的要求格式化数据。

### 修改analyzeDrawingWithAI函数

1. **更新请求体构建**：将用户的Base64编码图片包含在请求体的content字段中，使用多模态模型支持的格式
2. **添加图片格式验证**：确保传递的图片数据是有效的Base64编码字符串
3. **保持其他逻辑不变**：保留现有的错误处理、重试机制等

### 修改后的请求体结构

```typescript
const requestBody = {
  model: ARK_CHAT_MODEL,
  messages: [
    {
      role: 'system',
      content: HTP_SYSTEM_PROMPT
    },
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: '请分析以下房树人绘画作品，生成完整的心理分析报告。'
        },
        {
          type: 'image_url',
          image_url: {
            url: imageData // 直接传递Base64编码的图片
          }
        }
      ]
    }
  ],
  max_tokens: 1500,
  temperature: 0.6
};
```

## 验证步骤

1. **重启开发服务**：停止当前运行的服务，重新执行`npm run dev`
2. **测试图片上传**：在CanvasPage.tsx页面绘制房子、树和人，然后点击"完成绘制"按钮
3. **检查API调用**：查看浏览器控制台，确认请求体中包含了图片数据
4. **验证分析结果**：确认能够成功获取智能体的分析结果

## 预期结果
- 用户绘制的图片能够正确传递给智能体
- 智能体能够成功分析图片并返回结果
- 不再出现"InvalidParameter: 请求缺少必要参数或参数无效，请检查后重试"的错误

## 注意事项
- 确保传递的Base64编码图片包含正确的格式前缀（如`data:image/png;base64,`）
- 注意图片大小限制，避免传递过大的图片导致请求失败
- 保持请求体的格式与Doubao-1.5-vision-pro多模态模型的要求一致
