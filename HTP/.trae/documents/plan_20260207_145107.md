## 问题分析

用户指出内容应该是API# Role后台给，前台不应该有# Role。当前的问题是：

1. **后端硬编码了提示词**：在user消息中包含了详细的分析要求
2. **智能体配置也有提示词**：导致两者冲突
3. **智能体仍然输出不需要的标题**：没有按照后台配置的要求执行

## 解决方案

### 修改后端代码（`backend/server.js`）

#### 1. 简化user消息
- **移除硬编码的提示词**：从user消息中删除详细的分析要求
- **只提供基本信息**：只包含图片和用户的年龄、性别信息
- **让智能体自由发挥**：完全按照后台配置的提示词执行

#### 2. 保持请求结构
- **不添加system消息**：因为提示词应该由智能体后台配置提供
- **保持现有的请求格式**：确保与火山方舟API兼容

#### 3. 优化错误处理
- **保持现有的错误处理逻辑**：确保系统稳定运行
- **添加JSON解析验证**：确保智能体返回的内容是有效的

### 具体修改内容

1. **修改请求数据**：
   ```javascript
   const requestData = {
     model: VOLC_CONFIG.analysisEndpointId, // 用分析接入点ID
     stream: false,
     messages: [
       {
         role: 'user',
         content: [
           { type: 'text', text: `${age}岁，${gender}` },
           { type: 'image_url', image_url: { url: finalImageUrl } }
         ]
       }
     ],
     // 限制Token防止输出过长
     max_tokens: 4096
   };
   ```

2. **简化提示词**：只提供最基本的用户信息，不包含任何分析要求

3. **保持错误处理**：确保系统稳定运行

### 测试验证

1. **重启后端服务器**：
   - 停止当前运行的后端服务器
   - 重新启动后端服务器

2. **重启前端服务器**：
   - 停止当前运行的前端服务器
   - 重新启动前端服务器

3. **测试分析流程**：
   - 上传画作进行分析
   - 检查分析报告是否不再包含不需要的标题
   - 验证报告内容是否符合智能体后台配置的要求
   - 确认前端能够正确解析和显示报告内容

## 预期效果

- 智能体完全按照后台配置的提示词执行
- 不再输出"一、元素特征分析"、"二、心理状态评估"等不需要的标题
- 分析报告内容符合后台配置的要求
- 前端能够正确解析和显示报告内容
- 系统运行稳定，无错误
