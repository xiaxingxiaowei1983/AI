# 实现前后端分离架构

## 问题分析
当前系统直接在前端调用火山方舟API，存在以下问题：
1. 前端暴露敏感信息（API Key）
2. 前端传输大量提示词，消耗Token
3. 可能出现跨域和404错误

## 解决方案
实现前后端分离架构，前端只调用后端接口，后端负责调用火山方舟API。

### 修改内容

1. **更新前端.env文件**：
   - 只配置后端接口地址，不存储任何火山相关的敏感信息

2. **更新后端.env文件**：
   - 配置火山相关的敏感信息
   - 配置后端服务端口

3. **修改前端htpAnalysisService.ts文件**：
   - 调用后端接口，不再直接调用火山API
   - 只传递图片Base64、年龄、性别等变量

4. **修改后端server.js文件**：
   - 实现HTP分析接口
   - 实现文生图接口
   - 调用火山方舟API时使用智能体ID

### 修改后的代码

#### 1. 前端.env文件

```ini
# 前端.env（仅配置后端接口地址，同源无跨域）
VITE_BACKEND_BASE_URL=http://localhost:3000/api/htp
# （无需配置火山相关地址/ID/Key，全由后端处理）
```

#### 2. 后端.env文件

```ini
# 火山方舟核心配置（敏感信息，仅后端存储）
ARK_API_KEY=你的火山方舟专属API Key（无空格/无引号）
ARK_ANALYSIS_AGENT_ID=bot-20260205114157-98szj（HTP分析智能体ID）
ARK_IMAGE_AGENT_ID=bot-20260205152840-4qvqt（文生图智能体ID）
ARK_API_BASE_URL=https://ark.cn-beijing.volces.com/api/v3（火山基础地址，北京地域）

# 后端服务配置
PORT=3000（后端服务端口，和前端.env的3000对应）
```

#### 3. 前端htpAnalysisService.ts文件

```typescript
const BACKEND_BASE_URL = import.meta.env.VITE_BACKEND_BASE_URL;

// 前端仅传变量，无提示词/Key
export async function completeHTPWorkflow(
  imageBase64: string,
  userInfo: { age: number; gender: string }
) {
  try {
    // 1. 调用后端分析接口（传图片+年龄+性别）
    const analysisResponse = await fetch(`${BACKEND_BASE_URL}/analyze`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ imageBase64, ...userInfo })
    });
    const { report } = await analysisResponse.json();

    // 2. 调用后端文生图接口（传报告+年龄）
    const imageResponse = await fetch(`${BACKEND_BASE_URL}/generate-image`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ report, age: userInfo.age })
    });
    const { imageUrls } = await imageResponse.json();

    // 3. 返回结果（报告+插画URL）
    return { report, imageUrls };
  } catch (error) {
    console.error('流程失败:', error);
    throw error;
  }
}
```

#### 4. 后端server.js文件

```javascript
require('dotenv').config();
const express = require('express');
const axios = require('axios');
const cors = require('cors');

const app = express();
const PORT = process.env.PORT || 3000;

// 中间件：允许跨域（前端localhost:5173调用后端3000）
app.use(cors());
app.use(express.json({ limit: '20mb' })); // 支持大Base64数据

// 火山配置（从后端.env读取）
const VOLC_CONFIG = {
  apiKey: process.env.ARK_API_KEY,
  analysisAgentId: process.env.ARK_ANALYSIS_AGENT_ID,
  imageAgentId: process.env.ARK_IMAGE_AGENT_ID,
  chatUrl: `${process.env.ARK_API_BASE_URL}/chat/completions`,
  imageUrl: `${process.env.ARK_API_BASE_URL}/images/generations`
};

// 接口1：HTP分析（调用智能体A）
app.post('/api/htp/analyze', async (req, res) => {
  try {
    const { imageBase64, age, gender } = req.body;

    // 后端调用火山智能体A（带提示词）
    const response = await axios.post(
      VOLC_CONFIG.chatUrl,
      {
        model: VOLC_CONFIG.analysisAgentId, // 用分析智能体ID
        stream: false,
        messages: [
          { 
            role: 'user', 
            content: [
              { type: 'text', text: `绘画者：${age}岁 | ${gender}，请按智能体配置分析` },
              { type: 'image_url', image_url: { url: imageBase64 } }
            ]
          }
        ]
      },
      {
        headers: {
          'Authorization': `Bearer ${VOLC_CONFIG.apiKey}`,
          'Content-Type': 'application/json'
        }
      }
    );

    res.json({ report: response.data.choices[0].message.content });
  } catch (error) {
    console.error('分析接口失败:', error);
    res.status(500).json({ error: '分析失败' });
  }
});

// 接口2：文生图（调用智能体B）
app.post('/api/htp/generate-image', async (req, res) => {
  try {
    const { report, age } = req.body;

    // 后端调用火山智能体B（带文生图提示词）
    const response = await axios.post(
      VOLC_CONFIG.imageUrl,
      {
        model: VOLC_CONFIG.imageAgentId, // 用文生图智能体ID
        prompt: `报告：${report}，绘画者${age}岁，请按智能体配置生成3张疗愈插画`,
        n: 3,
        size: '223:100'
      },
      {
        headers: {
          'Authorization': `Bearer ${VOLC_CONFIG.apiKey}`,
          'Content-Type': 'application/json'
        }
      }
    );

    const imageUrls = response.data.data.map(item => item.url);
    res.json({ imageUrls });
  } catch (error) {
    console.error('文生图接口失败:', error);
    res.status(500).json({ error: '插画生成失败' });
  }
});

// 启动后端服务
app.listen(PORT, () => {
  console.log(`后端服务运行在 http://localhost:${PORT}`);
});
```

## 预期结果
- 前端不再暴露敏感信息
- 前端只传递必要的变量，减少Token消耗
- 后端调用火山方舟API时使用智能体ID，避免404错误
- 完整流程：用户上传图片→前端传变量→后端调用智能体→返回报告 + 3 张插画

## 注意事项
- 需要确保智能体已发布且绑定正确接入点
- 需要确保后端.env文件中的配置正确
- 需要确保前端.env文件中的后端接口地址与后端服务端口对应
- 需要先启动后端服务，再启动前端服务
