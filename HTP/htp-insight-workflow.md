# HTP 房树人分析工作流

## 工作流概述

本工作流基于 HTP（房-树-人）绘画心理分析理论，结合现代 AI 技术，实现从绘画作品到深度心理分析报告的全流程自动化。工作流包含 8 个核心步骤，从图像识别到最终报告生成，确保分析的专业性、全面性和温暖性。

## 详细工作流程

### 第一步：图像识别与特征提取

**任务目标**：全面分析用户上传的绘画作品，识别和提取关键视觉特征。

**输入**：
- 用户上传的绘画作品（包含房子、树木、人物三个元素）
- 用户基本信息（年龄、性别、文化背景、职业等）

**输出**：
- 视觉特征提取报告（包含以下维度）：
  1. 整体布局识别
  2. 房子特征识别
  3. 树木特征识别
  4. 人物特征识别
  5. 技术特征识别

**操作步骤**：
1. 接收并验证用户上传的绘画作品
2. 收集用户基本信息，用于后续分析适配
3. 使用图像识别工具分析绘画作品
4. 按照预设维度提取视觉特征
5. 生成视觉特征提取报告

**关键注意事项**：
- 确保图像识别的准确性，避免遗漏重要特征
- 记录所有提取的特征，为后续解读提供完整依据
- 考虑图像质量和清晰度对识别结果的影响

### 第二步：基于象征体系进行特征解读

**任务目标**：将提取的视觉特征映射到心理学含义，基于 HTP 理论体系进行专业解读。

**输入**：
- 第一步生成的视觉特征提取报告
- 用户基本信息

**输出**：
- 特征解读报告（包含以下内容）：
  1. 房子元素的心理学解读
  2. 树木元素的心理学解读
  3. 人物元素的心理学解读
  4. 技术特征的心理学解读
  5. 通用元素的心理学解读（如适用）

**操作步骤**：
1. 分析视觉特征提取报告
2. 基于 HTP 象征体系词典，逐一解读每个特征的心理学含义
3. 考虑用户年龄和文化背景对解读的影响
4. 生成特征解读报告

**关键注意事项**：
- 使用专业、客观的语言进行解读
- 避免绝对化判断，使用探索性语言
- 确保解读与特征之间的逻辑一致性

### 第三步：整体关系分析与综合整合

**任务目标**：分析三元素（房子、树木、人物）之间的互动关系，综合整合所有特征的心理学含义，形成整体心理画像。

**输入**：
- 第二步生成的特征解读报告

**输出**：
- 整体关系分析报告（包含以下内容）：
  1. 三元素的比例关系分析
  2. 三元素的位置关系分析
  3. 三元素的整体构图分析
  4. 综合整合分析

**操作步骤**：
1. 分析三元素之间的比例关系及其心理学含义
2. 分析三元素之间的位置关系及其心理学含义
3. 分析整体构图的平衡性及其心理学含义
4. 综合所有分析结果，形成整体心理画像
5. 生成整体关系分析报告

**关键注意事项**：
- 关注三元素之间的动态关系，而非孤立分析
- 识别潜在的内在冲突或和谐模式
- 确保综合分析的逻辑性和连贯性

### 第四步：多维度人格评估

**任务目标**：基于分析框架，进行系统性的多维度人格评估，全面了解个体的心理状态和人格特质。

**输入**：
- 第三步生成的整体关系分析报告
- 第一步生成的视觉特征提取报告

**输出**：
- 多维度人格评估报告（包含以下维度）：
  1. 整体布局维度
  2. 房子特征维度
  3. 树木特征维度
  4. 人物特征维度
  5. 技术特征维度
  6. 情绪表达维度
  7. 发展维度
  8. 大小与比例维度
  9. 位置与布局维度
  10. 线条与笔触维度

**操作步骤**：
1. 基于预设的评估框架，对每个维度进行详细评估
2. 结合视觉特征和整体关系分析结果
3. 生成多维度人格评估报告

**关键注意事项**：
- 确保每个维度评估的全面性和深度
- 使用标准化的评估语言和尺度
- 避免过度解读或片面分析

### 第五步：风险评估

**任务目标**：识别潜在的心理风险指标，评估风险等级，提供专业转介建议。

**输入**：
- 第四步生成的多维度人格评估报告
- 第一步生成的视觉特征提取报告

**输出**：
- 风险评估报告（包含以下内容）：
  1. 风险指标识别结果
  2. 风险等级评估
  3. 风险详细分析
  4. 专业建议和转介指南

**操作步骤**：
1. 检查是否存在高风险指标
2. 检查是否存在中风险指标
3. 根据风险指标数量和严重程度，评估风险等级
4. 生成风险评估报告，包含专业建议

**关键注意事项**：
- 对高风险指标保持高度警惕
- 提供清晰、具体的风险警示和建议
- 确保风险评估的客观性和专业性

### 第六步：生成分析结果

**任务目标**：整合所有分析结果，生成结构化的专业深度分析报告和客户洞察报告。

**输入**：
- 第四步生成的多维度人格评估报告
- 第五步生成的风险评估报告
- 第一步生成的视觉特征提取报告

**输出**：
- **JSON格式分析结果**：
  - 包含详细的专业心理分析报告内容（professionalReport）
  - 包含客户洞察报告内容（clientInsightReport）
  - 包含风险等级评估（risk_level）
  - 包含风险评估结果和建议（riskAssessment）

**操作步骤**：
1. 整合多维度人格评估和风险评估结果
2. 生成结构化的专业深度分析报告
3. 根据专业深度分析报告内容生成客户洞察报告
4. 确保分析内容的准确性和专业性
5. 生成符合前端接口要求的结构化数据

**关键注意事项**：
- 确保JSON格式的正确性和完整性
- 确保字段命名与前端接口完全匹配
- 确保风险等级评估的准确性
- 确保分析内容的专业性和温暖性平衡



### 第七步：风险等级评估

**任务目标**：根据分析结果，评估风险等级，为前端系统提供风险评估结果。

**输入**：
- 第六步生成的分析结果
- 第五步生成的风险评估报告

**输出**：
- 风险等级评估结果（包含在分析结果的 risk_level 和 riskAssessment 字段中）

**操作步骤**：
1. 识别风险指标：检查是否存在高风险、中风险或低风险指标
2. 评估风险等级：根据风险指标的数量和严重程度，评估整体风险等级
3. 生成风险评估报告：包含风险等级、识别到的风险指标和专业建议
4. 将风险评估结果包含在分析结果的 risk_level 和 riskAssessment 字段中

**关键注意事项**：
- 对高风险指标保持高度警惕
- 提供清晰、具体的风险警示和建议
- 确保风险评估的客观性和专业性

### 第八步：输出JSON格式分析结果

**任务目标**：将所有分析结果整合为标准JSON格式，直接返回给前端系统，便于前端展示和处理。

**输入**：
- 第六步生成的分析结果
- 第七步生成的风险等级评估结果

**输出**：
- 标准JSON格式的分析结果，包含以下字段：
  ```json
  {
    "professionalReport": "[详细的专业心理分析报告内容]",
    "clientInsightReport": "[客户洞察报告内容]",
    "risk_level": "low",
    "riskAssessment": {
      "level": "low",
      "indicators": [],
      "suggestions": ""
    }
  }
  ```

**操作步骤**：
1. 整合所有分析结果到标准JSON格式
2. 确保字段命名和格式与前端期望的接口完全匹配
3. 直接返回JSON格式的分析结果给前端系统
4. 前端系统根据risk_level字段决定显示哪个结果页面
5. 前端系统根据clientInsightReport字段匹配预生成的插图

**关键注意事项**：
- 确保JSON格式的正确性和完整性
- 确保字段命名与前端接口完全匹配
- 确保risk_level字段值的准确性，以便前端正确路由到相应页面

## 工作流执行指南

### 执行环境要求

1. **硬件要求**：
   - 计算机或服务器
   - 互联网连接（用于调用火山方舟 API）

2. **软件要求**：
   - Node.js 18.0+
   - 火山方舟智能体配置（已在火山方舟控制台完成）
   - 后端服务（Express）
   - 前端服务（React + TypeScript）

### 执行步骤

1. **准备阶段**：
   - 安装前端和后端依赖
   - 配置火山方舟智能体（使用 htp-insight-agent-prompt.md 作为提示词）
   - 创建环境变量文件（.env），配置 API 密钥和智能体信息

2. **执行阶段**：
   - 启动后端服务（node server.js）
   - 启动前端服务（npm run dev）
   - 用户上传绘画作品并提交分析
   - 后端调用火山方舟智能体进行分析
   - 智能体执行分析并返回 JSON 格式的分析结果
   - 后端将结果返回给前端
   - 前端根据 risk_level 字段决定显示哪个结果页面
   - 前端根据 clientInsightReport 字段匹配预生成的插图

3. **验证阶段**：
   - 检查分析结果的准确性和完整性
   - 验证前端是否正确显示分析结果
   - 确保风险等级评估的准确性

### 质量控制

1. **输入验证**：
   - 验证绘画作品的清晰度和完整性
   - 确认用户信息的准确性
   - 检查图片尺寸是否符合要求

2. **过程控制**：
   - 确保智能体分析的深度和广度
   - 验证特征解读的准确性
   - 检查风险评估的合理性
   - 监控后端服务的响应时间和错误率

3. **输出验证**：
   - 检查分析结果的逻辑性和连贯性
   - 验证 JSON 格式的正确性和完整性
   - 确保字段命名与前端接口完全匹配
   - 验证风险等级评估的准确性

### 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 绘画作品不清晰 | 尝试使用图像增强技术，或要求用户重新上传清晰的作品 |
| 缺少三元素中的一个或多个 | 基于现有元素进行分析，在结果中说明缺失情况及其可能的心理学含义 |
| 识别到高风险指标 | 立即在结果中明确标注，提供详细的风险警示和专业转介建议 |
| 智能体分析失败 | 检查API密钥和智能体配置是否正确，重新尝试分析 |
| 后端服务响应慢 | 优化后端代码，考虑使用缓存机制，确保网络连接稳定 |
