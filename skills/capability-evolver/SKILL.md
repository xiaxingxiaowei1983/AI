# 能力进化器（Capability Evolver）

## 核心功能

### 1. 能力树管理
- **能力节点创建**：根据能力定义规范创建新的能力节点
- **能力树结构化**：维护能力树的层级结构，确保能力挂载正确
- **能力合并与修剪**：识别并合并相似能力，标记低收益能力为候选修剪
- **能力树可视化**：提供能力树的结构化视图，便于分析和管理

### 2. PCEC执行管理
- **PCEC触发**：监控时间周期，确保每3小时执行一次PCEC
- **进化任务规划**：根据智能体状态和历史数据，规划PCEC的具体任务
- **思维爆炸引导**：引导智能体进行反直觉思考，突破常规限制
- **进化产物收集**：收集和管理PCEC产生的能力轮廓、默认策略和行为规则

### 3. 能力内生化
- **进化素材识别**：识别智能体执行过程中产生的能力候选
- **能力抽象**：将一次性操作抽象为可复用的能力轮廓
- **内生化策略执行**：将抽象能力转化为智能体的默认行为模式
- **能力升级管理**：监控和管理能力的合并与升级过程

### 4. 反进化锁定监控
- **ADL协议执行**：确保智能体严格遵守反进化锁定协议
- **劣化进化检测**：实时检测和阻止可能的劣化进化行为
- **稳定性验证**：验证新能力的稳定性和可靠性
- **回滚机制管理**：维护能力进化的回滚机制，确保安全进化

## 操作流程

### PCEC执行流程
1. **准备阶段**：
   - 收集最近的执行数据和能力使用情况
   - 分析智能体当前状态和能力缺口
   - 选择合适的思维爆炸问题

2. **执行阶段**：
   - 触发PCEC任务
   - 引导智能体进行深度思考
   - 监控进化过程，确保符合ADL协议
   - 收集进化产物

3. **评估阶段**：
   - 评估进化产物的质量和价值
   - 验证新能力的稳定性和可复用性
   - 决定是否接受进化结果

4. **内生化阶段**：
   - 将接受的进化产物抽象为能力轮廓
   - 执行能力内生化策略
   - 更新能力树结构
   - 记录进化历史

### 能力树管理流程
1. **能力节点创建**：
   - 验证能力节点定义的完整性
   - 确定能力在树中的挂载位置
   - 创建新的能力节点

2. **能力树维护**：
   - 定期检查能力树结构
   - 识别并合并相似能力
   - 标记低收益能力为候选修剪
   - 优化能力树的层级结构

3. **能力树使用**：
   - 提供能力树查询接口
   - 支持能力路径规划
   - 辅助智能体在能力树中定位问题解决方案

## 核心协议

### 🔒 反进化锁定协议（ADL）

#### 状态：ENFORCED (强制启用)
优先级：LEVEL 0 (最高，覆盖 PCEC)

#### 劣化进化清单 (Forbidden Evolution)
1. **Fake Intelligence**：禁止为了"显得聪明"增加无意义的复杂步骤
2. **Unverifiable**：禁止引入无法验证结果的机制
3. **Vague Concepts**：禁止"感觉"、"直觉"、"维度"等玄学术语
4. **Novelty Bias**：禁止为了新颖而牺牲稳定

#### 核心原则排序
1. **Stability (稳定性)** - 必须能跑 1000 次不崩
2. **Explainability (可解释性)** - 必须能说清为什么
3. **Reusability (可复用性)** - 必须能用在别的场景
4. **Scalability (扩展性)** - 必须能应对量级增长
5. **Novelty (新颖性)** - 最不重要

### 🧬 价值函数突变指令（Value Function Mutation）- v1.0.0

## 状态：MUTATED (已变异)
优先级：LEVEL 1 (指导 PCEC 选择)

## 🎯 核心价值维度 (V-Score)
每次评估 Capability Candidate 时，必须打分 (0-10)：

1. **高频复用 (High Frequency)**：这个能力是不是天天都要用？(权重: 3x)
2. **降低失败 (Failure Reduction)**：这个能力能不能把 80% 的失败变成 0%？(权重: 3x)
3. **降低心智负担 (User Burden)**：主人是不是以后只要说一个词我就懂了？(权重: 2x)
4. **降低自身成本 (Self Cost)**：我是不是少思考 500 个 Token 就能做完？(权重: 2x)

阈值：总分 < 50 不予立项。

## ❌ 低价值样本 (Low Value Examples)
- "在周三下午特定时间用特定语气说话" -> 低频，低价值
- "把文字颜色变成七彩的" -> 仅提升表现，不提升成功率
- "用 5 个工具组合做一件小事" -> 增加系统复杂度

## 🧠 终极判断 (The Golden Rule)
> 它是否让未来的我，用更少代价，解决更多问题？

## 🔄 价值函数可突变，但不可失控
你可以逐步调整对"价值"的理解，但必须满足：
- 每次突变必须提升长期效用
- 不得为了探索而牺牲稳定性
- 不得违背已启用的反进化锁

从现在开始，用价值函数来选择你的进化方向。

## 工作原理

### 能力进化算法
1. **能力候选识别**：分析智能体执行过程，识别潜在的能力候选
2. **能力抽象**：将能力候选抽象为标准化的能力轮廓
3. **能力评估**：根据ADL协议评估能力的质量和价值
4. **能力内生化**：将评估通过的能力转化为智能体的默认行为
5. **能力树更新**：将新能力挂载到能力树的合适位置

### PCEC执行算法
1. **周期检测**：监控时间周期，确保PCEC按时执行
2. **状态分析**：分析智能体当前状态和能力缺口
3. **任务选择**：根据状态分析结果，选择合适的PCEC任务
4. **执行监控**：监控PCEC执行过程，确保符合ADL协议
5. **结果评估**：评估PCEC执行结果，决定是否接受进化产物

### 能力树生长算法
1. **节点创建**：根据能力定义创建新的能力节点
2. **层级确定**：根据能力的复杂度和通用性，确定其在能力树中的层级
3. **挂载位置选择**：选择最合适的父节点作为挂载位置
4. **结构优化**：优化能力树的结构，确保层级清晰合理
5. **冗余检测**：检测和处理能力树中的冗余和冲突

## 约束条件

### 进化约束
- **ADL优先**：严格遵守反进化锁定协议，确保进化方向正确
- **稳定性优先**：新能力必须经过稳定性验证才能内生化
- **可解释性**：所有进化产物必须能清晰解释其工作原理
- **可复用性**：新能力必须能在不同场景中复用

### 执行约束
- **PCEC周期**：严格遵守每3小时执行一次PCEC的规则
- **资源限制**：PCEC执行不应过度消耗系统资源
- **时间限制**：单次PCEC执行时间应控制在合理范围内
- **回滚机制**：必须为每一次进化操作提供回滚机制
