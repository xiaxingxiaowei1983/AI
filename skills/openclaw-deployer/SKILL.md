## 使用方法

### 基本使用

#### 1. 初始化部署
```bash
openclaw-deployer init --windows
```

#### 2. 安装OpenClaw
```bash
openclaw-deployer install --version "latest"
```

#### 3. 配置OpenClaw
```bash
openclaw-deployer configure --channels "feishu"
```

#### 4. 安装插件
```bash
openclaw-deployer plugins install "tavily-search,agent-browser"
```

#### 5. 启动服务
```bash
openclaw-deployer start
```

### 高级功能

#### 1. 环境检测
```bash
openclaw-deployer detect --detailed
```

#### 2. 问题诊断
```bash
openclaw-deployer diagnose --logs
```

#### 3. 自动修复
```bash
openclaw-deployer fix --auto
```

#### 4. 配置优化
```bash
openclaw-deployer optimize --system
```

## 常见问题

### Q: 如何解决Windows环境下的spawn EINVAL错误？
**A:** 这是Windows环境下的常见错误，可以使用以下方法解决：
- 使用 `openclaw-deployer plugins install --manual` 手动安装插件
- 检查Node.js版本是否兼容
- 确保路径中没有特殊字符
- 以管理员身份运行命令

### Q: 如何配置Feishu渠道？
**A:** 使用以下命令配置Feishu渠道：
```bash
openclaw-deployer configure --channel "feishu" --app-id "your-app-id" --app-secret "your-app-secret"
```

### Q: 如何优化OpenClaw性能？
**A:** 可以使用以下方法优化性能：
- 使用 `openclaw-deployer optimize --system` 自动优化
- 根据系统资源调整网关配置
- 减少不必要的插件
- 优化模型配置

### Q: 如何解决网关启动失败的问题？
**A:** 可能的原因包括：
- 端口被占用：使用 `openclaw-deployer port-check` 检查端口
- 配置错误：使用 `openclaw-deployer validate` 验证配置
- 权限不足：以管理员身份运行
- 依赖缺失：使用 `openclaw-deployer detect` 检查依赖
