# Skill 2：用户验证脚本（Validation Kit）

## 目标
用最少成本验证三件事：
1) 用户是否真的有这个痛点（不是你想象的）
2) 用户是否愿意为解决方案付出（时间/授权/钱）
3) 你们的方案是否可行（不会一上来就被平台/习惯打死）

> 这一步的意义：避免你做完才发现“没人用”或“用户不愿意授权/不愿意付费”。

## 何时调用（触发条件）
- 你已经有 Skill 1 的一句话定位 v1
- 你准备写 PRD 之前（必跑）
- 你不确定需求是否真实、或者纠结功能取舍时

## 输入（固定表单）
- 产品名：
- 一句话定位（来自 Skill 1）：
- 目标用户（具体到人群）：____
- 目标场景（发生在什么时候/哪里）：____
- 你最担心的 3 个假设（来自 Skill 1）：____
- 你希望验证的形态：小程序 / H5 / 两者
- 是否涉及敏感权限（录音/位置等）：是/否

## 前置约束（硬规则）
- 访谈对象必须是真实目标用户（不是朋友夸你“很好”那种）
- 每次验证必须有“证据记录”（哪怕是 3 行总结）
- 任何结论都要能落到“继续/停止/转向”三选一

## 验证对象与样本（小白版建议）
- 最小样本：**5 人**（够你发现 80% 的方向性问题）
- 更稳样本：**10 人**
- 每个人控制在 10–15 分钟内，不要聊成咨询

## 三个关键假设（必须写清楚）
> 每个假设都要能判定“成立/不成立”。

- H1（痛点强度）：用户是否“经常遇到”这个问题？频率≥__ 次/周
- H2（解决意愿）：用户是否愿意为解决方案付出代价？
  - 例如：愿意授权录音/愿意花 __ 分钟/愿意付费 __ 元/月
- H3（方案可行）：用户能否理解并完成你设计的最小闭环？

## 访谈脚本（10 题以内，固定顺序）
> 你照着问就行，不用临场发挥。

1) 你最近一次遇到【痛点】是什么时候？发生了什么？  
2) 这件事你通常怎么解决？用什么工具？  
3) 你现在的解决方式哪里最难受？（时间/麻烦/不准/贵/隐私）  
4) 这个问题一周大概出现几次？（频率）  
5) 如果有一个工具能【你的收益】，你最希望它先解决哪一步？  
6) 你能接受为了用它做哪些事？（授权录音/注册/付费/等待）  
7) 你不能接受什么？（底线）  
8) 你愿意在什么情况下付费？（一次性/订阅/按次）  
9) 你觉得它要做到什么程度，你才会推荐给别人？  
10) 如果今天就有一个最简版本，你愿意试用吗？为什么？

## “展示原型”的方法（不做也行，但很省时间）
- 你只要拿一张纸/一张PPT画出：
  - 首页 → 关键按钮 → 结果页
- 问用户：你会点哪里？你觉得下一步会发生什么？
- 目的：验证“用户理解成本”，而不是验证 UI 好不好看

## 记录模板（每访谈 1 人填一次，3 分钟搞定）
- 用户编号：
- 角色/人群特征：
- 真实场景（1 句）：
- 现有解决方式：
- 最大痛点（1 句）：
- 是否愿意使用：是/否（原因）
- 是否愿意授权/付费：是/否（底线）
- 你学到的 1 个关键结论：

## 判定规则（继续 / 停止 / 转向）
> 这是本 skill 最核心的“省 token”部分：把决策写死，避免反复纠结。

### 继续（Go）条件（满足 ≥2 条就算 Go）
- G1：≥60% 受访者痛点频率高（例如每周≥2次）
- G2：≥60% 表示愿意试用最简版本
- G3：≥40% 愿意付出代价（授权录音/付费/注册至少一种）
- G4：多数人能理解你的最小闭环（不用你解释一堆）

### 停止（Stop）条件（满足任一条就 Stop）
- S1：大多数人不觉得这是问题（痛点不成立）
- S2：大多数人有更简单的替代方式且很满意
- S3：关键代价无法接受（例如绝大多数不愿意授权录音，而你必须录音）

### 转向（Pivot）条件（满足任一条就 Pivot）
- P1：痛点成立但你方案不对（用户想要的不是你做的那个）
- P2：用户愿意用但不愿意付费（考虑变现方式或渠道）
- P3：用户愿意付费但不愿意授权（考虑改成交互方式/形态）

## 输出格式（固定：Validation 结果单）
- Validation Kit：PASS / FAIL（这里 PASS 的意思是“已完成验证并得出结论”）
- 访谈人数：__
- 结论：继续 / 停止 / 转向
- 最强证据（3 条）：
  - E1：
  - E2：
  - E3：
- 用户最关心的 Top3：
  - T1：
  - T2：
  - T3：
- 你要调整的方向（若转向）：____

## 完成标准（本 skill 完成的判定）
- 至少完成 5 个真实用户访谈（或等价的真实证据）
- 有明确结论：继续/停止/转向（三选一）
- 输出 3 条“证据”，能支撑结论（不是感觉）
