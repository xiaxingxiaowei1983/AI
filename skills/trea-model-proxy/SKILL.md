## 使用方法

### 基本使用

#### 1. 安装SKILL
```bash
openclaw plugins install trea-model-proxy
```

#### 2. 配置智能体
在智能体的配置文件中添加以下配置：

```json
{
  "model": {
    "provider": "trea",
    "useInternalModel": true
  }
}
```

#### 3. 启动智能体
```bash
openclaw agent --agent [agent-name] --model "trea:internal"
```

### 高级配置

#### 1. 性能优化配置
```json
{
  "model": {
    "provider": "trea",
    "useInternalModel": true,
    "optimization": {
      "enableCache": true,
      "cacheSize": 100,
      "enableStreaming": true
    }
  }
}
```

#### 2. 会话管理配置
```json
{
  "model": {
    "provider": "trea",
    "useInternalModel": true,
    "session": {
      "maxHistory": 50,
      "enableSummary": true,
      "summaryThreshold": 20
    }
  }
}
```

## 能力节点定义

### 1. Trea模型调用器
- **能力名称**：调用Trea内置模型
- **输入条件**：
  - 提示词内容
  - 会话历史（可选）
  - 模型参数（可选）
- **输出结果**：
  - 模型生成的响应
  - 响应状态
  - 处理时间
- **成功前提**：
  - Trea环境正常运行
  - 输入格式正确
- **失败边界**：
  - Trea环境不可用
  - 输入内容过长
- **执行逻辑**：
  1. 检测Trea环境
  2. 封装请求
  3. 发送请求到Trea内置模型
  4. 接收并处理响应
  5. 返回处理结果

### 2. 智能体启动器
- **能力名称**：启动免密钥智能体
- **输入条件**：
  - 智能体名称
  - 智能体配置（可选）
- **输出结果**：
  - 智能体启动状态
  - 智能体实例
  - 启动日志
- **成功前提**：
  - 智能体配置正确
  - Trea环境可用
- **失败边界**：
  - 智能体配置错误
  - Trea环境不可用
- **执行逻辑**：
  1. 加载智能体配置
  2. 注入Trea模型代理能力
  3. 启动智能体
  4. 验证启动状态
  5. 返回启动结果

### 3. 配置生成器
- **能力名称**：生成免密钥配置
- **输入条件**：
  - 智能体类型
  - 运行环境信息
- **输出结果**：
  - 免密钥配置文件
  - 配置说明
  - 验证结果
- **成功前提**：
  - 提供有效的智能体类型
  - 运行环境可检测
- **失败边界**：
  - 智能体类型不支持
  - 运行环境异常
- **执行逻辑**：
  1. 检测运行环境
  2. 根据智能体类型生成配置
  3. 验证配置有效性
  4. 输出配置文件和说明

## 常见问题

### Q: 为什么智能体启动失败？
**A:** 可能的原因包括：
- Trea环境未正常运行
- 智能体配置错误
- 系统资源不足
- 网络连接问题

**A:** 可以尝试以下方法：
- 启用流式响应
- 减少会话历史长度
- 优化提示词长度
- 启用缓存功能

### Q: 智能体可以使用哪些Trea内置模型？
**A:** 目前支持Trea提供的所有内置模型，包括但不限于：
- 通用对话模型
- 代码生成模型
- 内容创作模型
- 数据分析模型

### Q: 如何验证配置是否正确？

```bash
openclaw agent --agent [agent-name] --test
```
