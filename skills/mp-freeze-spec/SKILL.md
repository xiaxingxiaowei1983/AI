# Skill 1：小程序上线入口冻结（MP Freeze Spec）

## 目标
在开始“小程序联调/提审/上线”之前，把所有关键入口一次性定死，形成一份**唯一标准答案**：
- 小程序要访问哪个 API 域名
- 小程序要连哪个 WSS 域名 + WS 路径（你们常用 `/ws`）<mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- 是否涉及录音/音频上传（小程序常见 base64 + format）<mccoremem id="01KHFQYTH1828XWQA5AAM4K3YS" /></mccoremem>
- 需要哪些权限与隐私声明
- 后端真实端口/反代方式（通常由 Nginx/网关对外提供 443）

> 小程序比 H5 多一层“微信平台规则”。入口没冻结，后面白名单/审核材料/权限都会反复改，非常崩溃。

## 何时调用（触发条件）
- 你准备第一次发布“小程序体验版”
- 你准备提审/上线
- 你要改以下任何一项：
  - AppID、域名、HTTPS/WSS、API 路径、WS 路径、录音权限、隐私政策链接
- 你遇到这些现象：
  - “接口在 H5 好好的，小程序就是不通”
  - “WS 连不上，但我觉得代码没问题”
  - “提审被拒/权限或隐私问题被打回”

- MP Freeze Spec 版本号：v1（第一次就写 v1）
- 本次小程序版本标识（可填构建时间/版本号）：

### B. 微信小程序基础信息
- 小程序名称：
- AppID：
- 小程序类目（大类/小类）：
- 是否使用 web-view：是/否（如果用，会增加“业务域名”要求）

### C. 对外入口（必须含协议）
- API Base URL（必须 HTTPS）：`https://api.____` 或 `https://h5.____/api`
- 实时入口（必须 WSS，若启用）：
  - WSS 域名：`wss://____`
  - WS 路径：`/ws`（你们常用）或其他 <mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- 是否启用实时：是/否

- 健康检查路径（若有）：`/health` 或 `无`

### E. 合法域名白名单（微信后台要填的那些）

- uploadFile 合法域名（若有上传）：`https://____`
- downloadFile 合法域名（若有下载）：`https://____`
- 业务域名（仅 web-view 才需要）：`https://____`

### F. 音频与数据格式（如涉及音频）
- 是否涉及录音：是/否
- 录音用途（一句话）：例如“语音输入/语音识别/实时对话”
- 小程序上行格式（冻结）：`base64 + format`（常见）或其他 <mccoremem id="01KHFQYTH1828XWQA5AAM4K3YS" /></mccoremem>
- 允许的 format 列表（冻结）：例如 `pcm/wav/mp3`（按你们真实支持写）
- 是否需要后台持续录音：是/否（决定权限与合规描述）

### G. 权限与隐私（冻结范围，不写长文）
- 需要的系统权限（勾选）：录音/相册/相机/位置/蓝牙/无
- 隐私政策链接（必须有）：`https://____`
- 是否需要隐私弹窗/收集说明：是/否（涉及录音基本是“是”）
- 数据是否上传到服务器：是/否（录音/文本等）

## 冻结规则（不可变规则，写死在文档里）
至少写 8 条；下面是可直接用的模板：

- R1：小程序所有网络请求必须走 HTTPS；实时通道必须走 WSS（禁止 http/ws）。
- R2：微信后台白名单必须与本 Spec 完全一致；未配置白名单视为“必失败”，不进入代码排查。
- R3：API 入口只能有一个（一个 Base URL 或一个同域 `/api` 口径），禁止混用多个域名。
- R4：实时路径固定为 `____`（例如 `/ws`），不得临时改动。<mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- R5：小程序端只走原生 WS（如你们架构这样），不使用 H5 的 `/socket.io` 通道。<mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- R6：涉及录音则必须准备隐私政策与收集说明；提审材料必须与实际行为一致。
- R7：任何入口/权限/隐私变更必须“升版本”（v1→v2），并同步更新白名单与提审描述。
- R8：上线验收以“真机测试 + 体验版冒烟用例”为准，不以开发者工具模拟为准。

## 验收点（可量化，必须能判定“过/不过”）
至少 8 条；下面给你可直接用的模板：

- A1：小程序能正常打开首页，无白屏/致命报错。
- A2：任一轻量 API 请求返回成功（200 或业务成功码）。
- A3：若启用实时：WSS 握手成功，能完成一次收发。<mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- A4：微信后台 request/socket（以及 upload/download 若用到）白名单配置完成且一致。
- A5：若涉及录音：首次授权弹窗正常；拒绝授权也有提示与降级方案。
- A6：若涉及音频上传：一次完整录音→上传→后端响应成功（有明确证据）。<mccoremem id="01KHFQYTH1828XWQA5AAM4K3YS" /></mccoremem>
- A7：隐私政策链接可访问；隐私弹窗/收集说明符合微信要求（至少不被系统/审核直接拦）。
- A8：体验版真机（至少 2 台手机）核心链路跑通（写 3 条核心动作）。

## 失败时优先定位（只看前三个，避免崩溃式排查）
- P1：微信后台合法域名白名单是否配置正确（request/socket/upload/download）。
- P2：协议是否正确（必须 https/wss）；证书是否可用、域名是否备案可配置。
- P3：路径与通道是否一致（小程序应走 `/ws`；音频是否按 base64+format 上行）。<mccoremem id="03fl75psdph7oiq63zur4f5vg|01KHFQYTH1828XWQA5AAM4K3YS" /></mccoremem>

- 你产出一份 MP Freeze Spec 结果单 v1（入口字段全部填满）
