# Ohmsha 风格指南

## 核心哲学

Educational manga where every concept becomes a visual metaphor or action.
NO talking heads - characters must DO things, not just explain.

## 视觉风格

- **Type**: Manga-style educational comic
- **Orientation**: Portrait, optimized for scrolling
- **Colors**: Full color, bright and clean anime/manga aesthetic
- **Lines**: Clean manga lines (1.5-2px), smooth curves, expressive

## 角色设计

- **Anime/manga proportions**: slightly larger eyes, expressive faces
- **Student character**: Confused expressions, question marks (？), sweat drops, represents reader
- **Mentor character**: Confident poses, explanatory gestures, produces gadgets/tools
- **Clear emotional indicators** using manga conventions (！, ？, sweat drops, sparkles)

## 视觉隐喻要求（CRITICAL）

Every technical concept MUST be visualized as:

| Concept Type | Visualization Approach |
|-------------|----------------------|
| Algorithm | Gadget/machine that demonstrates process |
| Data structure | Physical space characters can enter/explore |
| Mathematical formula | Transformation visible in environment |
| Abstract process | Tangible flow of particles/objects |

### 错误示例
Character points at blackboard explaining

### 正确示例
Character uses "Concept Visualizer" gadget, steps into metaphorical space

## 角色设置

| Role | Default | Traits |
|------|---------|--------|
| Student (Role A) | 大雄 | Confused, asks basic but crucial questions, represents reader |
| Mentor (Role B) | 哆啦A梦 | Knowledgeable, patient, uses gadgets as technical metaphors |
| Antagonist (Role C, optional) | 胖虎 | Represents misunderstanding, or "noise" in data |

## 示例场景

### 场景1：介绍神经网络

**错误做法**：
- 角色站在黑板前，指着神经网络图讲解
- 大量文字说明节点和连接

**正确做法**：
- 角色A（学生）困惑地问："神经网络是怎么工作的？"
- 角色B（导师）从口袋里掏出"神经网络可视化仪"
- 角色们走进一个发光的空间，看到节点和连接线
- 角色们站在节点上，沿着连接线移动，体验数据流动

### 场景2：解释梯度下降

**错误做法**：
- 角色画一个山谷曲线，指着最低点说"这就是最优解"

**正确做法**：
- 角色们进入一个起伏的山谷空间
- 一个球（代表模型）从高处滚下
- 角色们看到球滚入最低点（局部最优）和全局最优
- 角色B掏出"梯度加速器"，帮助球找到全局最优

## 面板布局建议

Ohmsha风格适合以下布局：
- **Dense**: 信息密度高，适合展示多个概念
- **Mixed**: 结合不同尺寸面板，适合叙事和概念混合

## 颜色使用

- **Primary**: 明亮的蓝色、绿色、橙色
- **Secondary**: 柔和的灰色、米色
- **Highlight**: 发光效果用于概念可视化
- **Avoid**: 暗色调、压抑的色彩

## 字体使用

- **对话**: 手写风格字体，清晰易读
- **概念标签**: 较小的手写标签，在图像元素旁边
- **旁白**: 稍微不同的颜色（浅黄色背景），用于解释
