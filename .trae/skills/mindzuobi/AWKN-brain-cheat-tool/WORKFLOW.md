| `COZE_API_TOKEN` | Coze API 认证令牌 | 扣子生产环境自动提供 |

如果扣子默认环境变量不可用，也可以使用 skill_credentials 配置：

| 环境变量 | 说明 |
|---------|------|
| `COZE_COZE_WORKFLOW_7597050443159846931` | Coze Workflow API Token |

## 标准流程

### 第一步：书籍内容拆解

**调用文件/技能**：
- `scripts/document_reader.py`（如果用户上传文档文件）

**技能作用**：
- 读取PDF/DOCX/TXT/EPUB格式的书籍文件
- 提取文本内容并保存供后续使用

**智能体作用**：
- 深度拆解书籍核心内容
- 提取主题定位、核心观点（3-5个）、内容结构
- 提取精选金句（10-20条）、关键案例、数据和引用
- 推荐2-4种可视化风格

**输出内容**：
```
【书籍拆解报告】
- 主题定位
- 核心观点（3-5个）
- 内容结构
- 精选金句（10-20条）
- 关键案例（完整故事案例）
- 数据和引用

【风格推荐】
- 2-4种可视化风格推荐
- 每种风格的推荐理由和适合场景
```

**数据保存**：
- 原文内容（供第二步使用）
- 拆解报告（供所有后续步骤使用）

### 第二步：生成公众号爆款文章

**调用文件/技能**：
- 基于认知工程学和AWKN爆文逻辑生成（完整流程见 AWKN-brain-cheat-tool/SKILL.md 第二步）
- 参考：AWKN-brain-cheat-tool/references/viral-logic.md（爆文逻辑指南）

**技能作用**：
- 智能体严格遵循 AWKN-brain-cheat-tool/SKILL.md 第二步的7个阶段（Phase 1-7）执行：
  1. 内容分析与提炼（4个核心要素）
  2. 标题生成（7大公式）
  3. 正文结构（6模块说服术，1500字）
  4. 语气控制（朋友式的叛逆者）
  5. 检查优化（素材来自原文，准确性）
  6. 转发文案生成（约140字内，不被折叠）
  7. 摘要生成（100字内）

**智能体作用**：
- 基于第一步的拆解报告框架
- 访问原文提取丰富素材（故事案例、金句、数据、场景）
- 严格按 SKILL.md 第二步流程生成内容
- 确保所有素材来自原文，保证准确性

**输出内容**：
```
【标题选项】
- 选项1：xxx
- 选项2：xxx
- 选项3：xxx

【正文】
- 1500字标准版文章
- 钩子开场（150字）
- 痛点共鸣（150-200字）
- 价值承诺（100字）
- 核心内容（800-1000字）
- 金句提炼（3-5句用*斜体*标记）
- 赋权结尾（150字）

【转发文案】（约140字内，不被折叠）
- 精炼金句或核心观点
- 吸引点击，提高打开率

【摘要】（100字内）
- 概括文章核心内容
- 提高点击率
```

**数据保存**：
- 公众号文章内容（供第二步可选、第七步使用）

```
【总结】
```

## 扩展流程

### 第五步：创作知识漫画（可选）

**调用文件/技能**：
- `AWKN-comic` Skill

**技能作用**：
- 将书籍内容转化为教育漫画
- 支持8种风格、6种布局、3种尺寸
- 生成8-16页漫画
- 合并为PDF

- 确定漫画主题和页数（建议8-16页）
- 确定叙事方案（按时间线/按主题/按角色）

**输出内容**：
```
【知识漫画】
- 分镜文件：storyboard-[chosen-scheme].md
- 漫画页面：01-page-[slug].png ~ NN-page-[slug].png
- PDF文件：{topic-slug}.pdf

主题：[漫画主题]
风格：[选择风格]
布局：[选择布局]
页数：X页
```

**数据保存**：
- 漫画PDF（供第七步使用）

## 数据流转图

```
第一步：书籍内容拆解
├─ 输入：书籍文件/纯文本
├─ 调用：scripts/document_reader.py（如果是文档文件）
├─ 输出：拆解报告、原文内容
└─ 数据传递：→ 第二步、第三步、第五步、第六步

第二步：生成公众号爆款文章
├─ 输入：第一步的拆解报告、原文内容
├─ 智能体：严格遵循 SKILL.md 第二步7个阶段（Phase 1-7）生成文章
├─ 参考：AWKN-brain-cheat-tool/references/viral-logic.md（爆文逻辑指南）
├─ 输出：标题选项、正文、转发文案、摘要
└─ 数据传递：→ 第二步（可选）、第七步

第二步（可选）：标题优化大师
├─ 输入：第二步的公众号文章内容
├─ 调用：AWKN-miming-title（功能1）
├─ 输出：10个爆款标题选项
└─ 数据传递：→ 用户选择

第三步：生成知识可视化图片
├─ 输入：第一步的拆解报告
├─ 调用：scripts/image_generator.py（5次） - 系列图片
├─ 调用：scripts/image_generator.py（3次） - 公众号插图
├─ 调用：AWKN-cover-image（自动执行）
├─ 输出：5张系列图片、3张公众号插图、文章封面
└─ 数据传递：→ 第三步（可选）、第七步

第三步（可选）：生成图文正文
├─ 输入：第一步的拆解报告、第二步的文章、第三步的5张图片、3张公众号插图
├─ 调用：AWKN-miming-title（功能2）
├─ 输出：500字精炼正文
└─ 数据传递：→ 第七步（模式B）

第四步：生成视频脚本
├─ 输入：第一步的拆解报告
├─ 智能体：严格遵循 SKILL.md 第四步流程生成视频脚本
├─ 参考：AWKN-brain-cheat-tool/references/video-script-format.md（视频脚本格式）
├─ 输出：90秒视频脚本
└─ 数据传递：→ 用户使用

第五步（可选）：创作知识漫画
├─ 输入：第一步的拆解报告
├─ 调用：AWKN-comic
├─ 输出：漫画PDF
└─ 数据传递：→ 第七步（模式C）

第六步（可选）：生成文字卡片
├─ 输入：第一步的拆解报告
├─ 调用：AWKN-text-card
├─ 输出：3-10张文字卡片
└─ 数据传递：→ 用户使用

第七步（可选）：发布到微信公众号
├─ 输入：第二步的文章（模式A）、第三步的5张系列图片+3张公众号插图+正文（模式B）、第五步的PDF（模式C）
```

### 标准流程（第一步-第四步）

- **第三步的5张图片**：传递给第三步（可选）、第七步（模式B）

- **第二步（可选）的图文正文**：传递给第七步（模式B）
