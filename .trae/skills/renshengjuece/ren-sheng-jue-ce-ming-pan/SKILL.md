# 人生战略决策系统「ren-sheng-jue-ce-ming-pan」

**命运没有标准答案，但博弈有最优解**

人生战略决策系统 = 东方命理×博弈论算法 | 在不确定的世界寻找确定性

## 核心功能

**日常决策支持**：针对通勤出行、事业发展、人际关系等日常场景，分析暗藏的顺逆趋势，识别潜在风险点。

**时机精准判断**：针对换工作、谈合作、启动新项目等关键决策，识别最佳时间窗口，提供顺势而为的行动建议。

**大事战略咨询**：面对跳槽、婚恋、跨城发展等人生关键岔路口，提供吉凶预判和赛道选择建议。

---

## 技能定位

人生战略决策系统是**总入口/聚合技能**，负责：
- 收集用户信息
- 自动调用底层技术组件
- 综合翻译为现代通识输出
- 提供标准化决策分析报告

**系统架构**：
```
ren-sheng-jue-ce-ming-pan（前台 - 用户交互）
    ↓ 调用底层组件
    ├─→ bazi-paipan（后台 - 八字排盘）→ 输出：八字命盘JSON
    ├─→ zi-ping-zhen-quan（后台 - 格局分析）→ 输出：格局分析JSON
    └─→ da-liu-ren（后台 - 决策预测）→ 输出：预测信号JSON
         ↓ 综合翻译
    输出：标准化决策分析报告（前台显示）
```

**核心理念**：命运没有标准答案，但博弈有最优解。

## 操作步骤

### 步骤一：信息收集

**判断用户是否提供了完整信息**：

必需信息：
1. ✅ 出生日期（公历/农历）
2. ✅ 出生时辰（精确到分钟）
3. ✅ 出生城市（省市，用于真太阳时修正）
4. ✅ 性别（男/女）

可选信息：
5. ⚪ 当前问题或想了解的方向（如：事业发展、婚姻情感、投资理财）

**如果信息不完整，输出标准提示**：
```
要为您进行人生决策分析，需要以下信息：
1. 出生日期（公历或农历）
2. 出生时辰（精确到分钟）
3. 出生城市（省+市）
4. 性别

**示例**：
"我是韩梅梅，公历1990年3月15日8:20分出生，女，出生地上海，想了解未来三年的事业发展机遇和风险点。"
```

### 步骤二：调用八字排盘（后台）

**自动调用** `bazi-paipan/scripts/bazi_paipan.py`：

```bash
cd /workspace/projects/bazi-paipan && python scripts/bazi_paipan.py \
  --name="姓名" \
  --gender=1 \
  --year=1990 \
  --month=3 \
  --day=15 \
  --hour=8 \
  --longitude=121.47 \
  --is-solar
```

**获取后台输出**（JSON格式）：
```json

    "year_pillar": "庚午",

    "day_pillar": "甲申",
    "hour_pillar": "戊辰"
  },
  "day_gan": "甲",
  "day_zhi": "申",
  "shi_shen": {...},
  "wu_xing": {...}

```

### 步骤三：调用格局分析（后台）

**根据八字数据，参考** `zi-ping-zhen-quan` 的格局分析逻辑：

**获取后台输出**（JSON格式）：

### 步骤四：调用演算沙推（后台）

**根据用户提问，调用** `da-liu-ren/scripts/day_liu_ren.py` 进行时间窗口预测：

```bash
cd /workspace/projects/da-liu-ren && python scripts/day_liu_ren.py \
  --year=2024 \
  --month=3 \
  --day=15 \
  --hour=14 \
  --question="换工作" \

  --day_zhi="申"
```

```json
{

    "asymmetric_risk": "high",
    "character_duty": "neutral"

    "bai_hu": true,
    "zhu_que": false
```

### 步骤五：综合翻译与输出（前台）

**根据后台输出数据，参考以下参考文档进行翻译和格式化**：

1. **术语翻译**：见 [references/术语翻译规则.md](references/术语翻译规则.md)
   - 将"身强"翻译为"高资源型 / 独立型"
   - 将"正官格"翻译为"管理赛道"
   - 将"白虎"翻译为"突发危机 / 黑天鹅"

2. **标准化输出格式**：见 [references/标准化输出格式.md](references/标准化输出格式.md)
   - 根据用户问题类型（资产/契约/发展）自适应输出侧重点
   - 输出包含决策模型分析（系统稳健性、长期复利、非对称风险、信用评级）

**最终输出**：标准化决策分析报告

## 注意事项

### 文档归属规则

- **理论文档**（如《渊海子平》）归档至 `zi-ping-zhen-quan`，确保底层工具包含完整理论支撑
- **前台技能**（ren-sheng-jue-ce-ming-pan）只保留交互和翻译相关的参考文档

### 职责边界

- **前台技能**：负责用户交互、场景自适应、翻译输出
- **后台组件**：只负责技术输出（JSON），不输出到前台

### 语言风格

- **现代通识风格**：使用商业逻辑、战略思维、风险管理等专业语言
- **避免过度游戏化**：不使用Buff/Debuff等游戏术语
- **统一格式**：使用"命运没有标准答案，但博弈有最优解"作为核心理念表达
