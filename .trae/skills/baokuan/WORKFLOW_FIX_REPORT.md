# 工作流引导修复报告

## 问题诊断

### 用户反馈
"我明确说要写公众号爆款文章，写完后没有提示下一步"

### 根本原因
在 **场景2（公众号完整发布流程）** 中，各个技能之间缺乏明确的**下一步引导**，导致用户完成文章创作后，不知道该做什么。

虽然根目录的 SKILL.md 和 awcn-skills-guide.md 中定义了完整的工作流，但各个技能本身的 SKILL.md 在完成任务后，没有主动提示用户进入下一步。

---

## 修复内容

### 修复原则
- **主动引导**：每个技能完成任务后，主动提示用户下一步选项
- **场景感知**：根据使用场景（完整流程 vs 单独使用）提供不同的引导
- **灵活选择**：允许用户选择"继续完整流程"或"只使用当前服务"
- **友好语气**：保持友好的语气，不强迫用户继续

## 引导原则

### 1. 主动引导
- 每个技能完成任务后，主动提示下一步选项
- 不等待用户询问，直接提供可操作的选项

### 2. 场景感知
- **完整流程**：提示后续所有步骤，形成闭环
- **单独使用**：只提供相关服务，不强制引导

### 3. 灵活选择
- 提供多个选项，用户可以自由选择
- 提供"不需要"选项，让用户可以随时退出

### 4. 友好语气
- 使用 ✅、🎉 等表情符号，增加友好感
- 清晰的选项描述，让用户知道每个选项的作用
- 不强迫用户继续，尊重用户的选择
