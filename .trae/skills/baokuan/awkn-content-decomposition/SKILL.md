# 深度内容拆解

## 任务目标
- 本技能用于：一键将复杂内容系统化拆解，提炼核心价值和可执行方法论
- 核心价值：将"被动输入"转化为"主动认知体系"，建立可复用的知识框架
- 能力包含：
  - 核心框架提炼：1句话核心命题 + 3-5个核心观点 + 价值定位 + 适用场景
  - 详细内容拆解：每个章节/段落的底层逻辑、核心概念、方法论、实践建议、案例说明
  - 质量检查：确保拆解的完整性和准确性
- 触发条件：用户提供书籍名称、文章链接、视频内容，明确需要拆解

## 前置准备
- 无特殊依赖

## 操作步骤

### 一键生产流程（标准）

**第一步：内容理解与类型识别**
- 分析用户提供的内容类型（书籍/文章/视频）
- 识别内容的核心主题和目标受众
- 确定拆解深度（全书/特定章节/核心观点）

**第二步：核心框架提炼**
- **核心命题**：1句话概括全书/全文的核心观点
- **核心观点**：提炼3-5个核心观点（每个观点1-2句话）
- **核心价值**：说明这本书/文章的价值所在（解决了什么问题，提供了什么方法）
- **适用场景**：分析适合的人群和使用场景

**第三步：详细内容拆解（必须展开，不能简略）**

根据内容类型，展开以下维度：

**如果是一本书**：
1. 底层逻辑：揭示的核心原理、科学依据
2. 核心概念：关键概念的定义和解释
3. 方法论：具体的步骤、框架、模型
4. 实践建议：可执行的行动建议
5. 案例说明：具体的案例或例子
6. 总结升华：全书的核心启示

**如果是一篇文章**：
1. 中心思想：文章的核心论点
2. 论证逻辑：文章的论证过程
3. 关键论据：支撑论点的关键论据
4. 实践建议：给出的具体建议
5. 总结：文章的结论

**第四步：拆解质量检查**
- ✅ 是否包含核心框架（核心命题、核心观点、核心价值、适用场景）
- ✅ 是否包含详细内容拆解（底层逻辑、方法论、实践建议、案例）
- ✅ 内容是否详细展开（不能只有简短概括）
- ✅ 是否提供了可执行的实践建议

**第五步：输出标准化拆解报告**

### 输出格式

**标准输出格式**：

```
【内容拆解报告】

# 一、核心框架
- 核心命题：[1句话概括核心观点]
- 核心观点：
  1. [观点1]
  2. [观点2]
  3. [观点3]
- 核心价值：[价值所在]
- 适用场景：[适用人群和场景]

# 二、详细内容拆解

[根据内容类型，详细展开以下内容]

## 第X章：[章节标题]
### 底层逻辑
[揭示的核心原理、科学依据]

### 核心概念
[关键概念的定义和解释]

### 方法论
[具体的步骤、框架、模型]

### 实践建议
[可执行的行动建议]

### 案例说明
[具体的案例或例子]

[继续其他章节...]

# 三、总结升华
[全书/全文的核心启示和实践路线图]
```

## 注意事项
- 必须详细展开，不能只有简短概括
- 每个章节都要包含：底层逻辑、核心概念、方法论、实践建议、案例说明
- 实践建议必须可执行，不能只是理论
- 案例说明要具体，有可参考性
- 避免常见错误：只提炼观点缺少展开、只列举概念缺少解释、只有理论缺少实践

## 使用示例

**示例1：一键拆解一本书**
- 用户指令："帮我拆解《认知觉醒》这本书"
- 智能体执行：自动加载技能，按标准流程生成12个章节完整拆解
- 输出内容：核心框架 + 详细拆解（每个章节5个维度） + 总结升华

**示例2：拆解一篇文章**
- 用户指令："拆解这篇文章的深度思考"
- 智能体执行：提炼中心思想、论证逻辑、关键论据、实践建议

**示例3：拆解特定章节**
- 用户指令："帮我拆解《认知觉醒》第1章和第3章"
- 智能体执行：深度拆解指定章节，确保每个维度都详细展开

## 核心亮点

✨ **一键生产**：提供书籍名称、文章链接或内容，自动生成完整的拆解报告

✨ **深度拆解**：每个章节都包含5个维度的详细展开，不是简单概括

✨ **标准化输出**：统一的输出格式，可直接用于知识库、内容创作、学习笔记

✨ **可执行性**：每个拆解都包含可执行的实践建议，从理论到落地

✨ **质量保证**：内置质量检查清单，确保拆解的完整性和准确性

✨ **实战验证**：内置完整拆解示例，可直接参考
