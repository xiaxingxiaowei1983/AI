# Git SSH 密钥配置计划

## 执行步骤

### 1. 检查本地现有密钥
- 查看 `.ssh` 目录是否存在
- 检查是否已有可用的 SSH 密钥对

### 2. 生成新的 SSH 密钥
- 执行 `ssh-keygen -t ed25519 -C "xiaxingxiaowei1983@gmail.com"` 命令
- 确认默认保存路径
- 选择是否设置安全密码短语

### 3. 启动 SSH 代理并添加密钥
- 启动 SSH 代理进程：`eval "$(ssh-agent -s)"`
- 将生成的私钥添加到代理：`ssh-add ~/.ssh/id_ed25519`

### 4. 获取公钥内容
- 查看生成的公钥内容：`cat ~/.ssh/id_ed25519.pub`
- 复制完整的公钥字符串

### 5. 上传公钥到 Git 平台
- 指导用户登录 GitHub/GitLab/Gitee 等平台
- 导航到 SSH 密钥设置页面
- 粘贴公钥并保存

### 6. 验证连接
- 测试与 Git 平台的连接：`ssh -T git@github.com`（或其他平台）
- 确认连接成功

## 注意事项
- 确保使用正确的邮箱地址
- 保护好私钥文件的安全性
- 按照平台要求正确粘贴公钥内容
- 验证连接时确认主机指纹
