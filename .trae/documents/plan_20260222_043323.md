# Git SSH 密钥配置详细计划

## 执行步骤

### 1. 检查本地现有密钥
- 确认 `.ssh` 目录存在
- 检查是否已有密钥文件

### 2. 生成新的 SSH 密钥（非交互式）
- 使用非交互式命令生成密钥，避免等待输入
- 命令：`ssh-keygen -t ed25519 -C "xiaxingxiaowei1983@gmail.com" -f "$HOME\.ssh\id_ed25519" -N ""`

### 3. 启动 SSH 代理（Windows 特定）
- 在 Windows 上启动 SSH 代理服务
- 命令：`Start-Service ssh-agent`

### 4. 添加密钥到 SSH 代理
- 将生成的私钥添加到代理
- 命令：`ssh-add "$HOME\.ssh\id_ed25519"`

### 5. 验证密钥生成
- 检查生成的密钥文件
- 命令：`Get-ChildItem -Path "$HOME\.ssh"`

### 6. 查看并复制公钥内容
- 读取公钥文件内容
- 命令：`Get-Content "$HOME\.ssh\id_ed25519.pub"`
- 复制完整的公钥字符串

### 7. 上传公钥到 Git 平台
- 指导登录 GitHub/GitLab/Gitee 等平台
- 导航到 SSH 密钥设置页面
- 粘贴公钥并保存

### 8. 验证连接
- 测试与 Git 平台的连接
- 命令：`ssh -T git@github.com`（或其他平台）
- 确认连接成功

## 注意事项
- 使用非交互式命令避免沙箱环境的输入问题
- 确保正确处理 Windows 路径格式
- 保护好私钥文件的安全性
- 按照平台要求正确粘贴公钥内容
- 验证连接时确认主机指纹
