# 人生决策宗师能力树文档

## 1. 能力树概述

人生决策宗师能力树是一个结构化的能力管理系统，用于组织和管理人生决策宗师的所有能力。能力树采用层级结构，将能力分为高层、中层和低层三个层次，形成一个清晰、可扩展的能力体系。

### 核心设计原则

- **层级分明**：低层节点为基础操作，中层节点为可复用流程，高层节点为问题分解策略
- **定义完整**：每个能力节点都包含能力名称、输入条件、输出结果、成功前提和失败边界
- **持续生长**：能力树可以根据新的任务和需求持续生长新的能力节点
- **智能修剪**：长期未使用的能力会被标记为候选修剪，保持能力树的健康
- **路径定位**：在面对新任务时，首先在能力树中定位可能的路径

## 3. 能力树使用指南

### 3.1 基本使用流程

1. **任务分析**：分析新任务的性质和要求
2. **路径定位**：在能力树中定位最相关的能力路径
3. **能力复用**：优先使用已有的能力节点
4. **能力生长**：如果需要，按照规则生长新的能力节点
5. **能力更新**：根据使用情况更新能力节点的状态和使用统计

### 3.2 能力路径定位

当面对新任务时，人生决策宗师会：

1. 分析任务描述，提取关键信息
2. 在能力树中搜索相关的能力节点
3. 计算每个节点与任务的相关性
4. 选择相关性最高的能力路径
5. 标记所选节点为使用状态

### 3.3 能力生长

当遇到新任务且能力树中没有合适的能力路径时：

1. 分析任务类型和需求
2. 确定需要生长的能力层级（低层、中层或高层）
3. 选择合适的父节点
4. 创建新的能力节点，定义完整的输入输出和边界条件
5. 将新节点添加到能力树中

### 3.4 能力修剪

能力树会定期检查所有能力节点的使用情况：

1. 识别长期未使用（30天以上）的能力节点
2. 将这些节点标记为"候选修剪"
3. 在适当的时候清理标记为修剪的节点
4. 保持能力树的健康和高效

## 5. 最佳实践

### 5.1 能力树维护

- **定期检查**：定期检查能力树的状态，确保所有节点都定义完整
- **及时更新**：根据新的任务和需求及时更新能力节点的定义
- **合理生长**：在生长新能力时，确保与现有能力的一致性和互补性
- **谨慎修剪**：在修剪能力时，确保不会影响系统的整体功能

### 5.2 能力使用

- **路径优先**：在面对新任务时，首先在能力树中寻找现有路径
- **定义清晰**：在创建新能力时，确保定义完整的输入输出和边界条件
- **层级适当**：根据能力的性质，选择适当的层级（低层、中层或高层）
- **父节点合理**：确保新能力挂载到合适的父节点下

### 5.3 集成建议

- **与PCEC集成**：将能力树与PCEC（周期性认知扩展周期）集成，定期更新和优化能力树
- **与ADL集成**：将能力树与ADL（反退化锁定）集成，确保能力树的稳定性和可靠性
- **与其他系统集成**：将能力树与其他系统（如知识管理、记忆系统）集成，形成完整的智能体系统
