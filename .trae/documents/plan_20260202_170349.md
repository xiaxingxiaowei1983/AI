## 页面逻辑分析与智能体接口标准

### 当前系统工作流程

**输入阶段**

* 用户输入：出生年份、月份、日期、小时、分钟、日历类型（阳历/农历）、性别

* 数据格式：JSON对象包含所有出生信息

**计算阶段**（后端逻辑）

* 根据出生信息计算八字四柱（年柱、月柱、日柱、时柱）

* 确定日主和五行属性

* 判断格局类型

* 生成年度关键词和描述

**输出阶段**

* 生成高光日历列表（全年）

* 每个高光日包含：日期、类型、评分、描述、行动指南

### 智能体接口标准

**接口URL**: `POST /api/decision-analysis`

**请求格式**:

```json
{
  "birthInfo": {
    "year": 1995,
    "month": 1,
    "day": 1,
    "hour": 12,
    "minute": 0,
    "isLunar": false,
    "gender": "male"
  },
  "userQuestion": "用户输入的问题（可选）"
}
```

**响应格式**:

```json
{
  "userBazi": {
    "yearPillar": "乙亥",
    "monthPillar": "丙寅",
    "dayPillar": "壬子",
    "hourPillar": "丙午",
    "dayMaster": "壬",
    "dayMasterWuxing": "水",
    "pattern": "食神格"
  },
  "annualKeyword": "创造",
  "patternDescription": "食神生财，创意无限，适合发挥才华，创造新价值",
  "highlightDays": [
    {
      "date": "2026-02-12",
      "dateStr": "2026-02-12",
      "type": "财运亨通",
      "score": 85,
      "description": "关注投资机会，但需谨慎决策",
      "actionGuide": "建议稳健投资，控制风险敞口"
    }
  ],
  "aiAnswer": "如果用户有问题，AI智能体基于八字命盘给出决策建议"
}
```

### 字段说明

**birthInfo** - 出生信息

* year: 年份（1970-2050）

* month: 月份（1-12）

* day: 日期（1-31）

* hour: 小时（0-23）

* minute: 分钟（0-59）

* isLunar: 是否农历（true/false）

* gender: 性别（"male"/"female"）

**userQuestion** - 用户问题（可选，字符串）

**userBazi** - 用户八字命盘

* yearPillar: 年柱（如"乙亥"）

* monthPillar: 月柱（如"丙寅"）

* dayPillar: 日柱（如"壬子"）

* hourPillar: 时柱（如"丙午"）

* dayMaster: 日主（如"壬"）

* dayMasterWuxing: 日主五行（如"水"）

* pattern: 格局（如"食神格"）

**annualKeyword** - 年度关键词

* 单个词，如"创造"、"财富"、"智慧"等

**patternDescription** - 年度描述

* 详细解释年度关键词的含义

**highlightDays** - 高光日历数组

* date: 完整日期（YYYY-MM-DD格式）

* dateStr: 日期字符串（用于显示）

* type: 高光日类型

  * "财官双美" - 财运和事业双重提升

  * "贵人相助" - 获得贵人帮助

  * "桃花盛开" - 感情运势旺盛

  * "事业高升" - 事业运势上升

  * "财运亨通" - 财运顺畅

  * "智慧开启" - 智力提升

  * "平安顺遂" - 平安顺利

* score: 评分（0-100整数）

* description: 简短描述

* actionGuide: 行动指南

**aiAnswer** - AI回答（可选）

* 如果用户输入了问题，AI智能体需要基于八字命盘给出决策建议

* 格式：字符串，建议长度：100-500字

### 实现建议

1. **后端API开发**

   * 创建 `/api/decision-analysis` 接口

   * 接收出生信息和用户问题

   * 调用八字计算引擎

   * 生成高光日历

   * 如果有用户问题，调用AI智能体生成建议

2. **前端修改**

   * 在BirthInputModal中添加问题输入框

   * 在提交时将问题传递给后端

   * 在ResultDashboard中显示AI回答（如果存在）

3. **AI智能体选择**

   * 推荐使用支持中文的大语言模型

   * 如：通义千问、智谱AI、月之暗面等

   * 需要具备八字命理知识库

   * 输出格式需要符合上述JSON标准

4. **数据流**

   * 用户输入出生信息 → 后端计算八字 → 生成高光日历 → 返回给前端

   * 用户输入问题 → 后端调用AI智能体 → 生成决策建议 → 返回给前端

   * 前端合并两个响应显示给用户

这个标准可以让您开发一个完整的API接口，前端和后端都能按照这个规范进行对接。
