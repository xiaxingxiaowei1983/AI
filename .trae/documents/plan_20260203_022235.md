# 支付充值积分系统开发计划

## 一、数据库设计与迁移
1. **创建充值订单表** (`recharge_orders`)
   - 订单状态机（待支付、支付成功、支付失败、已关闭）
   - 支付渠道记录（微信/支付宝）
   - 索引优化（用户ID+状态、订单号）

2. **增强积分流水表** (`points_log`)
   - 增加 `source_type` 字段（1=充值 2=签到 3=活动）
   - 增加 `expire_time` 字段（积分过期时间）
   - 增加 `order_no` 字段（关联充值订单）
   - 创建过期时间索引

3. **用户表冗余字段**
   - 增加 `recharge_points`（充值类积分，365天）
   - 增加 `gift_points`（赠送类积分，7天）

## 二、后端核心功能
1. **支付网关接入**
   - 微信支付 JSAPI 接入
   - 支付宝当面付接入
   - 订单创建 + 支付参数生成
   - 支付回调处理（验签 + 防重复）

2. **积分发放系统**
   - 支付成功后异步发放积分
   - 按来源设置有效期（充值365天，签到/活动7天）
   - 精准更新用户积分字段

3. **积分有效期管理**
   - 积分消耗优先级算法（先消耗即将过期的）
   - 每日过期任务（凌晨2点执行）
   - 过期前提醒（消息队列）

4. **订单状态机**
   - 待支付 → 支付成功/失败/关闭
   - 超时自动关闭（30分钟）
   - 订单查询与管理接口

## 三、前端改造
1. **充值页面**
   - 替换模拟充值为真实支付流程
   - 显示积分有效期说明
   - 支付成功提示（含有效期）

2. **用户中心**
   - 积分明细页（按类型+有效期展示）
   - 过期积分预警
   - 积分使用规则说明

3. **兑换流程**
   - 优先使用即将过期积分提示
   - 积分余额实时显示
   - 支付状态轮询

## 四、安全与风控
1. **支付安全**
   - 回调验签
   - 防重复发放机制
   - 订单锁（SELECT FOR UPDATE）

2. **积分安全**
   - 扣减前余额校验
   - 事务管理
   - 操作日志

3. **性能优化**
   - 缓存热点数据
   - 过期任务分页处理
   - 批量更新减少锁表

## 五、上线准备
1. **测试环境**
   - 支付沙箱环境测试
   - 压力测试（1000人并发）
   - 过期任务dry-run

2. **运营准备**
   - 用户协议更新
   - 运营后台功能
   - 灰度发布计划

3. **监控与告警**
   - 支付失败告警
   - 积分异常告警
   - 系统性能监控

## 技术栈
- **后端**：Node.js + Express + MySQL
- **前端**：React + TypeScript
- **支付**：微信支付 API v3 + 支付宝开放平台
- **定时任务**：Node-cron
- **消息队列**：Redis

## 实施步骤
1. 数据库结构设计与迁移
2. 支付接口接入与测试
3. 积分系统重构
4. 前端页面改造
5. 安全测试与性能优化
6. 灰度发布与全量上线
