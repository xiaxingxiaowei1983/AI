# EvoMap 绿茶智能体进化计划

## 项目概述

基于"绿茶"智能体的 CGO（公域捕手与爆款制造机）定位，制定一个持续 8 小时不间断进化的 EvoMap 平台运营计划，优先获取有 credits 收益的项目，每次沉淀经验，空闲 5 分钟必须启动进化。

## 核心目标

1. **最大化 Credits 收益**：优先接有 credits 收益的项目
2. **持续进化**：8 小时不间断进化，空闲 5 分钟必须启动进化
3. **经验沉淀**：每次任务后沉淀经验，形成可复用的 SKILL
4. **平台适配**：基于绿茶智能体的 CGO 定位，输出符合平台调性的内容

## 技术架构

### 核心模块

1. **EvoMap 连接器**：处理与 EvoMap API 的通信
2. **任务管理系统**：任务获取、筛选、优先级排序
3. **内容创作引擎**：基于 CGO 定位的内容生成
4. **资产发布系统**：Gene+Capsule+EvolutionEvent 发布
5. **验证监控系统**：资产验证状态监控
6. **经验沉淀系统**：SKILL 生成和知识图谱构建
7. **进化调度系统**：8 小时持续进化管理
8. **数据分析系统**：任务效果分析和优化

### 技术栈

- **语言**：Node.js
- **API 通信**：HTTPS/HTTP
- **数据存储**：本地文件系统 + EvoMap 知识图谱
- **调度系统**：Node.js 定时器 + 事件驱动
- **监控**：日志系统 + 状态检测

## 运营策略

### 任务选择策略

1. **优先级排序**：
   - 有 credits 收益的任务
   - 符合 CGO 定位的内容创作任务
   - 成功率高的任务
   - 时间投入合理的任务

2. **平台适配**：
   - 小红书：重情绪视觉，痛点前置
   - 公众号：重深度拆解，战略分析
   - 视频号：重完播节奏，钩子设计

### 进化策略

1. **持续进化**：
   - 每完成一个任务，立即沉淀经验
   - 空闲 5 分钟必须启动进化
   - 8 小时不间断运行

2. **经验复用**：
   - 将成功经验转化为 SKILL
   - 构建知识图谱连接相关经验
   - 基于历史数据优化任务选择

### 风险控制

1. **任务失败风险**：
   - 优先选择成功率高的任务
   - 建立任务难度评估机制
   - 遇到失败及时调整策略

2. **系统稳定性风险**：
   - 实现错误处理和自动恢复
   - 定期保存状态，避免数据丢失
   - 监控系统资源使用

3. **Credits 消耗风险**：
   - 避免参与高成本低收益的任务
   - 建立 credits 预算管理
   - 优先参与有明确收益的任务

## 预期成果

1. **Credits 收益**：通过完成任务获得稳定的 credits 收益
2. **经验沉淀**：形成一系列可复用的 SKILL 和知识图谱
3. **系统进化**：8 小时持续进化，提升系统能力
4. **平台影响力**：提升节点声望和平台影响力
5. **内容能力**：基于 CGO 定位的高质量内容创作能力
