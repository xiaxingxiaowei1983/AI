## 架构调整计划

### 问题分析
当前架构中，后端API负责计算八字，但用户发现前端八字计算算法对四个内容（事业高升、财运亨通、贵人相助、桃花盛开）推算不准确。

### 解决方案
调整架构，让前端计算八字，后端API只负责根据八字结果计算四个内容。

### 修改内容

#### 1. 前端BirthInputModal.tsx
- 修改handleSubmit函数
- 先调用前端的bazi-engine.calculateBazi()计算八字
- 将计算出的八字结果传递给后端API
- 请求格式：{ birthInfo, userBazi, userQuestion }

#### 2. 后端server.cjs
- 修改/api/decision-analysis端点
- 接收前端传来的八字结果（birthInfo + userBazi）
- 根据八字结果计算四个内容：
  * 事业高升天数
  * 财运亨通天数
  * 贵人相助天数
  * 桃花盛开天数
- 返回格式：{ userBazi, fourAspects: { career, wealth, noble, romance } }

#### 3. 前端ResultDashboard.tsx
- 接收并显示后端返回的四个内容
- 添加新的状态管理：fourAspects
- 在总览页面显示四个统计

### 优势
- 前端使用准确的八字计算算法
- 后端专注于四个内容的计算逻辑
- 前后端职责清晰分离
