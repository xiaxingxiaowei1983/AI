# 人生决策宗师 - 价值函数突变指令文档

## 1. 概述

### 1.1 什么是价值函数突变指令

价值函数突变指令（Value Function Mutation）是人生决策宗师智能体的核心能力之一，它允许智能体基于内在价值函数来评估和选择能力进化方向，而不是平均对待所有潜在能力。

### 1.2 核心目标

- **智能进化选择**：基于价值函数评估，优先发展高价值能力
- **系统效率提升**：减少低价值能力的资源占用，优化系统性能
- **稳定性保障**：通过价值评估，确保新增能力能够提升系统整体稳定性
- **用户体验优化**：优先发展能够简化用户操作的能力
- **系统确定性增强**：通过价值评估，确保系统行为更加可预测

## 2. 价值函数核心组件

### 2.1 价值函数（ValueFunction）

**功能**：提供能力价值评估的核心逻辑，包含五个核心价值维度的评估机制。

**核心价值维度**：
- **复用频率潜力**：能力在不同场景中的复用可能性
- **对失败率的影响**：能力是否能降低系统失败率
- **是否减少用户认知负担**：能力是否简化用户操作和理解
- **是否减少自身推理/工具成本**：能力是否提高系统执行效率
- **是否提升系统级确定性**：能力是否使系统行为更加可预测

**使用示例**：
```javascript
const { valueFunction } = require('./capabilities/value-function');

// 评估能力
const evaluation = valueFunction.evaluateCapability(capability);
console.log('价值评分:', evaluation.score);
console.log('评估原因:', evaluation.reasons);
```

### 2.2 能力价值评估系统（CapabilityValueEvaluator）

**功能**：为智能体提供能力价值评估和管理功能，支持对现有能力和新能力的评估。

**主要功能**：
- 单个能力评估
- 批量能力评估
- 能力树评估
- 价值排行榜生成
- 进化队列管理

**使用示例**：
```javascript
const { capabilityValueEvaluator } = require('./capabilities/capability-value-evaluator');

// 评估能力树
const treeEvaluation = capabilityValueEvaluator.evaluateCapabilityTree();
console.log('评估完成:', treeEvaluation.count, '个能力');

// 生成价值排行榜
const ranking = capabilityValueEvaluator.generateValueRanking();
console.log('高价值能力:', ranking.highValueCount, '个');
```

### 2.3 低价值能力识别与管理（LowValueCapabilityManager）

**功能**：为智能体提供低价值能力识别、标记和管理功能。

**低价值能力类型**：
- **极端场景使用**：只在极端场景使用的能力
- **只提升表现不提升成功率**：只提升表现但不提升成功率的能力
- **无法跨任务迁移**：无法跨任务迁移的能力
- **增加系统复杂度**：会增加系统复杂度的能力

**使用示例**：
```javascript
const { lowValueCapabilityManager } = require('./capabilities/low-value-capability-manager');

// 识别低价值能力
const identification = lowValueCapabilityManager.identifyLowValueCapabilities(capabilities);
console.log('低价值能力:', identification.count, '个');

// 生成修剪建议
const suggestions = lowValueCapabilityManager.generateTrimSuggestions();
console.log('修剪建议:', suggestions.count, '个');
```

### 2.4 价值函数突变机制（ValueFunctionMutator）

**功能**：为智能体提供价值函数突变和管理功能，支持价值维度权重的调整。

**主要功能**：
- 执行价值函数突变
- 生成推荐权重
- 分析突变影响
- 生成突变计划
- 回滚机制

**使用示例**：
```javascript
const { valueFunctionMutator } = require('./capabilities/value-function-mutator');

// 生成推荐权重
const recommendation = valueFunctionMutator.generateRecommendedWeights('stabilityFocus');
console.log('推荐权重:', recommendation.recommendedWeights);

// 执行突变
const mutationResult = valueFunctionMutator.mutateValueFunction(recommendation.recommendedWeights, '稳定性优先');
console.log('突变结果:', mutationResult.success ? '成功' : '失败');
```

## 3. 价值函数使用指南

### 3.1 基本使用流程

1. **评估能力**：使用价值函数评估能力树中的所有能力
2. **生成排行榜**：基于评估结果生成价值排行榜
3. **管理进化队列**：根据价值排行榜管理能力进化队列
4. **识别低价值能力**：识别并标记低价值能力
5. **生成修剪建议**：为低价值能力生成修剪建议
6. **执行价值突变**：根据系统需求调整价值函数权重

### 3.2 能力评估流程

1. **输入能力信息**：提供能力的名称、描述、输入、输出、前提条件和失败边界
2. **价值维度评估**：评估能力在五个核心价值维度上的表现
3. **综合价值计算**：基于权重计算能力的综合价值分数
4. **低价值类型检查**：检查能力是否属于低价值类型
5. **生成评估报告**：生成详细的能力评估报告

### 3.3 进化队列管理

1. **评估能力树**：评估能力树中的所有能力
2. **筛选高价值能力**：筛选价值分数高于阈值的能力
3. **排序进化队列**：基于价值分数排序进化队列
4. **选择进化候选**：选择队列中的前几个能力作为进化候选
5. **执行能力进化**：对候选能力进行进化

## 4. 价值函数突变指南

### 4.1 突变场景

| 场景 | 描述 | 推荐权重调整 |
|------|------|------------|
| stabilityFocus | 稳定性优先 | 增加failureRateImpact和systemCertainty权重 |
| efficiencyFocus | 效率优先 | 增加reasoningCostReduction和cognitiveLoadReduction权重 |
| generalOptimization | 通用优化 | 平衡所有维度权重 |

### 4.2 突变约束

- **稳定性优先**：failureRateImpact权重不能低于当前值
- **避免极端变化**：单个维度权重变化不超过50%
- **保持平衡**：最大权重比不超过5:1

### 4.3 突变执行流程

1. **生成突变计划**：根据场景生成推荐权重
2. **分析突变影响**：分析突变对系统的影响
3. **验证突变方案**：验证突变方案是否符合约束
4. **执行突变**：执行价值函数突变
5. **监控系统状态**：监控突变后的系统状态
6. **必要时回滚**：如果突变导致问题，执行回滚

## 5. 低价值能力管理

### 5.1 低价值能力识别

- **基于价值分数**：价值分数低于0.3的能力
- **基于能力类型**：属于四种低价值能力类型之一
- **基于使用频率**：长期未使用的能力

### 5.2 低价值能力处理

| 价值分数 | 处理建议 |
|---------|--------|
| < 0.2 | 建议立即修剪 |
| 0.2-0.3 | 建议在下次维护时修剪 |
| 0.3-0.6 | 建议评估后决定 |
| > 0.6 | 不建议修剪 |

### 5.3 修剪操作

1. **评估修剪影响**：评估修剪对系统的影响
2. **执行修剪**：执行能力修剪操作
3. **验证系统状态**：验证修剪后的系统状态
4. **清理相关资源**：清理与能力相关的资源

## 6. 系统集成指南

### 6.1 与能力树集成

价值函数系统与能力树系统紧密集成，通过以下方式：

1. **能力评估**：评估能力树中的所有能力
2. **进化队列**：基于评估结果管理能力进化队列
3. **能力修剪**：识别并修剪低价值能力
4. **价值趋势分析**：分析不同级别能力的价值趋势

### 6.2 与ADL协议集成

价值函数系统遵循ADL（Anti-Degeneration Lock）协议的约束：

1. **稳定性优先**：在价值评估中，稳定性是重要的评估维度
2. **可解释性**：评估结果必须可解释
3. **避免玄学术语**：评估过程中避免使用模糊术语
4. **可验证性**：评估结果必须可验证

### 6.3 与PCEC集成

价值函数系统与PCEC（Periodic Cognitive Expansion Cycle）集成：

1. **进化方向选择**：为PCEC提供进化方向建议
2. **进化产物评估**：评估PCEC生成的进化产物
3. **进化验证**：验证进化是否符合价值函数要求

## 10. 故障排除

### 10.1 常见错误和解决方案

| 错误 | 原因 | 解决方案 |
|------|------|--------|
| 能力评估失败 | 能力信息不完整 | 确保提供完整的能力信息 |
| 突变执行失败 | 不符合约束条件 | 调整突变方案，符合约束条件 |
| 系统性能下降 | 低价值能力过多 | 清理低价值能力 |
| 进化队列过长 | 进化执行频率低 | 增加进化执行频率 |
| 评估结果不准确 | 价值维度权重不当 | 调整价值维度权重 |

### 10.2 系统恢复

1. **回滚价值函数**：如果突变导致问题，执行价值函数回滚
2. **恢复能力树**：如果修剪导致问题，恢复被修剪的能力
3. **重启系统**：如果系统出现严重问题，重启系统
4. **恢复备份**：如果所有方法都失败，恢复系统备份
