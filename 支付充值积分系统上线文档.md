# 支付充值积分系统上线文档

## 1. 项目概述

本项目实现了一个完整的支付充值积分系统，将原有的"点击即到账"的模拟充值替换为真实支付流程，并实现了充值积分（365天）与领取积分（7天）的精准生命周期管理。

### 核心目标
- 接入支付网关，实现真实支付流程
- 积分生命周期管理，按来源差异化设置有效期
- 支付风控，防止重复发放积分

## 2. 实现的功能

### 2.1 后端功能
- 充值订单创建接口
- 支付回调处理接口
- 订单状态查询接口
- 每日积分过期任务

### 2.2 前端功能
- 真实支付流程（创建订单 -> 发起支付 -> 轮询状态 -> 发放积分）
- 积分明细展示（区分充值积分和赠送积分）
- 积分有效期显示
- 即将过期积分提示
- 积分使用规则说明

### 2.3 积分系统功能
- 积分有效期管理（充值积分365天，其他积分7天）
- 积分消耗优先级（优先使用即将过期的积分）
- 积分过期检查和处理
- 管理员积分管理

## 3. 技术架构

### 3.1 前端
- React 18 + TypeScript

- Vite
- LocalStorage 存储用户数据

### 3.2 后端
- Node.js + Express

### 3.3 支付流程
- 前端调用后端创建订单接口
- 后端生成订单并返回支付参数

## 5. API接口

### 5.1 充值订单创建
- **接口**: POST /api/recharge/create-order
- **参数**:
  - amount: Number (充值金额，分)
  - pay_channel: String (支付渠道，wechat/alipay)
- **返回**:
  - order_no: String (订单号)
  - points_granted: Number (应得积分)
  - pay_params: Object (支付参数)

### 5.2 订单状态查询
- **接口**: GET /api/recharge/status
- **参数**:
  - order_no: String (订单号)
- **返回**:
  - status: Number (订单状态)
  - status_text: String (状态文本)
  - amount: Number (充值金额)
  - points_granted: Number (应得积分)
  - pay_channel: String (支付渠道)
  - paid_time: Date (支付时间)

### 5.3 支付回调
- **接口**: POST /api/recharge/notify
- **参数**:
  - order_no: String (订单号)
  - transaction_id: String (支付平台流水号)
  - pay_channel: String (支付渠道)
- **返回**:
  - success: Boolean (处理结果)
  - message: String (处理消息)
