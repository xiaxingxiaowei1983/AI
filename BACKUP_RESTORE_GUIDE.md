# 大宗师系统备份与恢复指南

## 1. 备份系统概述

本系统采用双保险备份策略：
- **本地备份**：存储在 `./backups` 目录，按星期分类
- **远程备份**：存储在 iCloud（可选）

### 备份触发条件
- 距离上一次备份超过24小时
- 文件大小变化超过10KB

## 2. 手动执行备份

### 方法一：运行备份脚本
```powershell
# 在项目根目录执行
node backup-system.js
```

### 方法二：通过定时任务（推荐）
系统已配置定时任务，会自动执行备份。

## 3. 恢复系统

### 方法一：运行恢复脚本
```powershell
# 在项目根目录执行
node restore-system.js
```
脚本会自动列出可用的备份，并默认恢复最新的备份。

### 方法二：手动恢复
1. 进入 `./backups` 目录
2. 选择对应星期的文件夹（如 `./backups/周一`）
3. 找到最新的备份文件（格式：`backup-时间戳.zip`）
4. 解压备份文件到项目根目录
5. 重启系统

## 5. 备份内容

备份文件包含以下目录：
- `./agents` - 智能体配置
- `./data` - 系统数据
- `./configs` - 系统配置
- `./skills` - 技能库
- `./waytoagi-knowledge` - 知识库
- `./waytoagi-community` - 社区资源

## 6. 紧急恢复流程

如果系统出现严重问题，请按以下步骤恢复：

1. **停止所有运行的服务**
   ```powershell
   # 停止所有相关进程
   Stop-Process -Name node -Force
   ```

2. **执行恢复脚本**
   ```powershell
   node restore-system.js
   ```

3. **重启系统**
   ```powershell
   # 启动大宗师智能体
   npm run start-master
   ```

4. **验证系统状态**
   - 检查智能体是否正常启动
   - 测试基本功能是否正常

## 7. 定期检查

建议每周检查一次备份状态：

1. 查看 `./backup-log.json` 了解备份历史
2. 确认备份文件大小合理
3. 验证远程备份是否成功

## 8. 故障排除

### 备份失败
- 检查磁盘空间是否充足
- 检查 iCloud 同步是否正常
- 查看 `backup-log.json` 中的错误信息

### 恢复失败
- 检查备份文件是否损坏
- 确保目标目录有写入权限
- 尝试使用不同的备份文件

## 9. 最佳实践

- **定期测试恢复**：每月至少测试一次恢复流程
- **保存重要配置**：对关键配置文件进行额外备份
- **监控备份状态**：设置备份状态通知
- **保持备份目录清洁**：定期清理过期备份
