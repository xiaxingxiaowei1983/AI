# 大脑作弊器 - 数据库接入详细执行计划

## 一、项目概述

### 目标
将"大脑作弊器"Web应用接入数据库系统，实现用户认证、脚本管理、思维模型动态加载、统计分析等功能。

### 技术栈
- **数据库**: PostgreSQL (通过 Drizzle ORM)
- **对象存储**: S3 兼容对象存储 (用于文件存储)
- **后端**: Next.js 16 + TypeScript
- **前端**: 现有的 HTML + Vanilla JavaScript

---

## 二、功能需求分析

### 2.1 核心功能

#### 1. 用户认证系统
- 用户注册（用户名、邮箱、密码）
- 用户登录（用户名/邮箱 + 密码）
- 会话管理（登录状态保持）
- 管理员账户（特殊权限）

**用户场景**：
```
用户访问页面 → 点击"登录" → 输入账号密码 → 登录成功
  → 可以查看历史记录、收藏脚本等
```

#### 2. 脚本管理功能
- 历史记录（用户生成的所有脚本）
- 脚本收藏（收藏重要脚本）
- 脚本分享（生成分享链接，可公开访问）
- 脚本删除（删除不需要的脚本）

**用户场景**：
```
用户生成脚本 → 自动保存到历史记录
  → 可以查看历史、收藏脚本、生成分享链接
  → 登录后跨设备访问历史记录
```

#### 3. 思维模型动态加载
- 从数据库动态加载思维模型
- 管理员可通过后台添加/编辑/删除模型
- 支持分类管理（创新&战略、产品&设计、用户&需求等）

**用户场景**：
```
用户访问"思维模型库" → 从数据库加载模型数据
  → 管理员登录后台 → 添加新模型 → 前端自动更新
```

#### 4. 数据统计与分析
- 脚本生成次数统计
- 热门思维模型排行
- 用户活跃度统计
- 文件上传统计

**管理员场景**：
```
管理员登录后台 → 查看数据统计
  → 了解平台使用情况、热门模型等
```

#### 5. 文件存储管理
- 支持文件上传（PDF、EPUB、TXT、图片）
- 文件临时存储（当天有效）
- S3 对象存储集成

**用户场景**：
```
用户上传PDF文件 → 文件上传到S3临时存储
  → 提取文本内容生成脚本 → 文件当天有效
```

#### 6. 管理员功能
- 管理员登录（特殊权限）
- API 接口管理（配置 2 个 API 接口）
- 思维模型管理（增删改查）
- 查看数据统计

**管理员场景**：
```
管理员访问 /admin → 登录后台
  → 管理 API 接口配置
  → 管理思维模型库
  → 查看数据统计
```

## 三、数据库表结构设计

### 3.1 表结构概览

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| `users` | 用户表 | id, username, email, password, role |
| `scripts` | 脚本表 | id, userId, title, name, dao, fa, shu, qi, li |
| `script_shares` | 脚本分享表 | id, scriptId, shareToken, isPublic |
| `script_favorites` | 脚本收藏表 | id, userId, scriptId |
| `thinking_models` | 思维模型表 | id, name, description, category, content |
| `usage_statistics` | 使用统计表 | id, userId, action, modelId, timestamp |
| `file_uploads` | 文件上传表 | id, userId, fileName, fileKey, expireAt |
| `api_configs` | API 配置表 | id, configKey, configValue, description |

### 3.2 详细表设计

#### 表 1: users - 用户表
```sql
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(128) NOT NULL UNIQUE,
  email VARCHAR(255) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) DEFAULT 'user', -- 'user' 或 'admin'
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP
);
```

#### 表 2: scripts - 脚本表
```sql
CREATE TABLE scripts (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  title VARCHAR(255),
  name TEXT,              -- 名：核心概念
  dao TEXT,               -- 道：底层逻辑
  fa TEXT,                -- 法：执行框架
  shu TEXT,               -- 术：具体方法
  qi TEXT,                -- 器：工具资源
  li TEXT,                -- 例：实战案例
  input_type VARCHAR(20), -- 输入类型：file/url/image
  input_content TEXT,     -- 原始输入内容
  model_used VARCHAR(100),-- 使用的思维模型
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP
);
```

#### 表 3: script_shares - 脚本分享表
```sql
CREATE TABLE script_shares (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  script_id VARCHAR(36) NOT NULL REFERENCES scripts(id),
  share_token VARCHAR(64) NOT NULL UNIQUE,
  is_public BOOLEAN DEFAULT TRUE,
  view_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 表 4: script_favorites - 脚本收藏表
```sql
CREATE TABLE script_favorites (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  script_id VARCHAR(36) NOT NULL REFERENCES scripts(id),
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(user_id, script_id)
);
```

#### 表 5: thinking_models - 思维模型表
```sql
CREATE TABLE thinking_models (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,        -- 模型名称
  description TEXT,                  -- 简短描述
  category VARCHAR(50) NOT NULL,     -- 分类：innovation/product/user/growth/thinking/analysis
  content JSONB,                     -- 详细内容（JSON格式）
  tags TEXT[],                       -- 标签数组
  usage_count INTEGER DEFAULT 0,     -- 使用次数
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP
);
```

#### 表 6: usage_statistics - 使用统计表
```sql
CREATE TABLE usage_statistics (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36) REFERENCES users(id),
  action VARCHAR(50) NOT NULL,       -- 动作类型：generate_script/view_model/favorite_script/share_script
  model_id VARCHAR(36) REFERENCES thinking_models(id),
  metadata JSONB,                    -- 额外信息
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 表 7: file_uploads - 文件上传表
```sql
CREATE TABLE file_uploads (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36),              -- 可以为空（未登录用户）
  file_name VARCHAR(255) NOT NULL,
  file_key VARCHAR(500) NOT NULL,   -- S3 对象存储的 key
  file_type VARCHAR(50),
  file_size INTEGER,
  expire_at TIMESTAMP NOT NULL,     -- 过期时间（当天23:59:59）
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 表 8: api_configs - API 配置表
```sql
CREATE TABLE api_configs (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  config_key VARCHAR(100) NOT NULL UNIQUE,
  config_value TEXT NOT NULL,
  description TEXT,
  is_active BOOLEAN DEFAULT TRUE,
  updated_at TIMESTAMP DEFAULT NOW()
);
```

## 五、用户体验流程

### 5.1 用户注册登录流程

```
1. 用户访问首页 → 点击"登录"按钮
   ↓
2. 打开登录模态框 → 点击"立即注册"
   ↓
3. 填写用户名、邮箱、密码 → 提交注册
   ↓
4. 注册成功 → 自动登录 → 页面刷新
   ↓
5. 显示用户名 → 可以访问个人功能
```

### 5.2 生成脚本流程（已登录用户）

```
1. 用户登录后 → 进入"大脑作弊器"模块
   ↓
2. 选择输入类型（文件/链接/截图）→ 上传或输入内容
   ↓
3. 点击"生成认知"按钮 → 显示进度条
   ↓
4. 后端生成脚本 → 保存到数据库（关联用户ID）
   ↓
5. 显示生成结果 → 自动保存到历史记录
   ↓
6. 用户可以查看历史、收藏、分享脚本
```

### 5.3 查看历史记录

```
1. 用户登录后 → 点击导航栏"历史记录"
   ↓
2. 显示脚本列表（按时间倒序）
   ↓
3. 点击某个脚本 → 查看详情
   ↓
4. 可以收藏、分享、删除脚本
```

### 5.4 脚本分享流程

```
1. 用户查看脚本详情 → 点击"分享"按钮

2. 生成分享链接 → 复制链接
```

```
   - 思维模型（增删改查模型）
```

## 六、文件存储方案

### 6.1 S3 对象存储集成

#### 上传文件流程
```
1. 用户上传文件 → 前端发送到 /api/upload
   ↓
2. 后端接收文件 → 上传到 S3 对象存储
   ↓
3. 获取 fileKey → 保存到 file_uploads 表
   ↓
4. 返回 fileKey → 前端使用
   ↓
5. 当天 23:59:59 → 自动清理过期文件
```

#### 文件存储结构
```
uploads/
├── {date}/           # 按日期分目录
│   ├── pdf/
│   ├── image/
│   └── text/
└── temp/
```

#### 临时清理机制
```javascript
// 定时任务：每天凌晨清理过期文件
async function cleanupExpiredFiles() {
  const expiredFiles = await db.file_uploads
    .where(expireAt < new Date())
    .get();
  
  for (const file of expiredFiles) {
    await s3Storage.deleteFile({ fileKey: file.fileKey });
    await db.file_uploads.delete({ id: file.id });
  }
}
```
