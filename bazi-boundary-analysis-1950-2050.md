# 八字算法边界日期和特殊情况分析（1950-2050）

## 1. 年份边界分析

### 1.1 立春时间分析
立春是八字年柱切换的关键时间点，需要特别关注每年立春前后的日期计算。

**关键年份立春时间：**
- 1950年：2月4日 17:00
- 1960年：2月5日 02:55
- 1970年：2月4日 14:47
- 1980年：2月5日 00:09
- 1990年：2月4日 10:14
- 2000年：2月4日 20:40
- 2010年：2月4日 06:52
- 2020年：2月4日 17:03
- 2030年：2月4日 03:08
- 2040年：2月4日 15:06
- 2050年：2月4日 01:18

**测试重点：**
- 立春前一天 vs 立春当天 vs 立春后一天
- 立春时间前后的小时跨越
- 跨年的年柱计算准确性

## 2. 月份边界分析

### 2.1 节气切换点
每个月的开始以节气为标志，需要验证节气前后的月柱计算。

**关键节气：**
- 寅月：立春
- 卯月：惊蛰
- 辰月：清明
- 巳月：立夏
- 午月：芒种
- 未月：小暑
- 申月：立秋
- 酉月：白露
- 戌月：寒露
- 亥月：立冬
- 子月：大雪
- 丑月：小寒

**测试重点：**
- 每个节气前后的日期
- 节气时间点的小时跨越
- 特殊节气如春分、秋分、夏至、冬至

## 3. 日边界分析

### 3.1 子时处理
子时（23:00-01:00）是日柱切换的关键时间，需要特别关注。

**测试重点：**
- 23:00-23:59：应属于次日的日柱
- 00:00-00:59：应属于当日的日柱  
- 01:00-01:00：应属于当日的日柱
- 跨日的子时计算

### 3.2 特殊日期
- **闰年2月29日**：如2000年、2004年、2008年、2012年、2016年、2020年、2024年、2028年、2032年、2036年、2040年、2044年、2048年
- **年末年初**：12月31日和1月1日
- **月份第一天和最后一天**

## 6. 特殊情况分析

### 6.1 边界日期组合
- **立春+闰年**：如2020年2月4日立春，2020年是闰年
- **立春+月末**：如某些年份立春在2月3日或4日
- **节气+月末**：如小寒在1月5日或6日
- **节气+月初**：如立夏在5月5日或6日

### 6.2 时间边界组合
- **子时+年末**：12月31日23:00-23:59
- **子时+月初**：1月1日00:00-00:59

### 6.3 算法边界情况
- **缓存边界**：缓存键的生成和管理
- **数据边界**：节气数据的边界年份（1950和2050）
