# OpenClaw 接入飞书完整教程
OpenClaw 原生支持飞书（Lark）接入，采用 WebSocket 长连接方案，**无需公网IP、无需配置回调地址**，全程10分钟即可完成配置，以下是官方标准接入流程。

## 一、前置准备
1. 已完成 OpenClaw 的部署与基础配置，确保服务可正常运行
2. 拥有飞书企业账号，建议使用**企业管理员权限**登录飞书开放平台，简化应用发布流程
3. 终端/服务器可正常访问 OpenClaw 服务与飞书开放平台

## 二、飞书开放平台核心配置（关键步骤）
### 1. 创建企业自建应用，获取核心凭证
- 访问飞书开放平台官网：https://open.feishu.cn/app ，使用企业账号登录
- 点击「创建企业自建应用」，填写应用名称（如 OpenClaw AI 助手）、应用描述，可自定义上传应用图标，完成创建
- 进入应用详情页，在**凭证与基础信息**页面，复制并妥善保存 **App ID**（格式为 `cli_xxx`）和 **App Secret**，后续配置必须使用，请勿泄露

### 2. 启用机器人能力
- 左侧导航栏找到「应用能力」，点击「添加应用能力」
- 在弹出的能力列表中选择**机器人**，点击「添加」，完成机器人能力启用
- 可按需配置机器人的名称、头像、欢迎语等展示信息，保存配置

### 3. 开通核心权限（必须配置，否则无法收发消息）
- 左侧导航栏进入「权限管理」页面，批量开通以下核心权限：
  - `im:message`：获取与发送单聊、群聊消息
  - `im:message:send_as_bot`：以机器人身份发送消息
  - `im:chat:readonly`：获取群组基本信息
  - `contact:user.id:readonly`：获取用户基本ID信息
- 权限开通后状态为「已申请」，随应用发布后自动生效

### 4. 配置事件订阅（核心关键，必须选长连接）
- 左侧导航栏进入「事件与回调」页面
- 在「回调配置」-「订阅方式」中，**必须选择「使用长连接接收回调」**（WebSocket 模式，无需公网IP、无需配置回调地址，是 OpenClaw 官方推荐的最简方案），点击保存
- 点击「添加事件」，在「消息与群组」分类中，选择**接收消息 `im.message.receive_v1`**，完成事件添加
- 无需配置加密密钥、校验 Token，长连接模式下会自动处理

### 5. 发布应用，完成飞书端配置
- 点击页面顶部的「创建版本」，填写版本号、更新说明，点击保存
- 点击「申请发布」，提交企业管理员审批；企业自建应用通常可自动通过，或联系企业管理员快速审批
- 只有发布完成的应用，才能在飞书客户端中找到并正常使用

## 三、OpenClaw 端飞书通道配置
提供3种配置方式，按需选择即可，**推荐使用前两种官方原生方式**。

### 方式一：安装向导添加（新安装用户首选）
刚完成 OpenClaw 初始安装后，系统会自动弹出安装向导，直接根据向导提示：
1. 选择「Feishu/飞书」渠道
2. 按提示粘贴之前保存的飞书 App ID 和 App Secret
3. 向导会自动完成配置与连接测试，无需额外手动操作

### 方式二：命令行交互式添加（已部署用户首选）
登录 OpenClaw 所在的服务器/终端，执行以下命令进入渠道添加流程：
```bash
openclaw channels add
```
1. 在交互式菜单中，选择 `Feishu/Lark (飞书)`，回车确认
2. 按提示依次粘贴飞书 **App ID** 和 **App Secret**，回车确认
3. 系统会自动校验凭证并完成配置，提示成功即可

### 方式三：进阶配置方式
#### 1. 配置文件方式
编辑 OpenClaw 核心配置文件 `~/.openclaw/openclaw.json`，添加飞书渠道配置：
```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "accounts": {
        "main": {
          "appid": "你的飞书App ID",
          "appsecret": "你的飞书App Secret"
        }
      }
    }
  }
}
```
保存文件后，执行重启网关命令即可生效。

#### 2. 可视化面板方式（云部署/面板用户）
- 进入 OpenClaw 可视化管理后台，找到「通道配置/消息平台配置」
- 选择「飞书」渠道，填入 App ID 和 App Secret
- 点击「保存/应用」，系统会自动重启服务完成配置

## 四、启动服务与测试验证
### 1. 重启网关，应用配置
终端执行以下命令，重启 OpenClaw 网关，让飞书配置生效：
```bash
# 重启网关
openclaw gateway restart

# 查看网关运行状态
openclaw gateway status

# 深度校验飞书通道状态
openclaw status --deep
```
当输出中 Feishu 通道显示 `ON` 和 `OK`，说明配置生效，与飞书服务器连接正常。

### 2. 消息测试与配对授权
1. 打开飞书客户端，在「工作台」找到你发布的 OpenClaw 应用，进入机器人对话界面
2. 给机器人发送任意一条测试消息（如“你好”）
3. 默认安全模式下，机器人会回复一个**配对码**，需要回到 OpenClaw 终端执行授权命令：
```bash
openclaw pairing approve 你的配对码
```
4. 授权完成后，机器人即可正常响应所有消息，支持单聊、群聊@机器人使用

## 五、常见问题排查
1. **机器人收不到消息/不回复**
   - 检查飞书应用是否已成功发布，核心权限是否全部开通并生效
   - 确认事件订阅选择了「长连接模式」，且已添加 `im.message.receive_v1` 事件
   - 检查 App ID 和 App Secret 是否填写正确，无多余空格或字符错误
   - 执行 `openclaw logs --follow` 查看实时日志，排查具体报错

2. **提示权限不足**
   - 回到飞书开放平台「权限管理」，确认所有核心权限都已开通，且已随版本发布生效
   - 确认飞书账号在应用的可用范围内，管理员已开启使用权限

3. **群聊中@机器人无响应**
   - 确认机器人已被添加到对应群聊
   - 确认 `im:chat:readonly` 权限已开通，事件订阅配置正确
