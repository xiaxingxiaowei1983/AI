# Skill 6：提审/发布流程冒烟（Submit & Release Smoke Test）

## 目标
在“提交审核/发布上线”之前，用最少步骤确认三件事：
1) 体验版在真机上能跑通核心功能（不是模拟器）
2) 版本/环境/域名都对（不是拿错包去提审）
3) 审核材料与路径清晰（审核员能复现）

> 很多人不是“代码有 bug”，而是“提审前没做体验版真机冒烟”，导致审核来回打回、浪费时间。

## 何时调用（触发条件）
- 每次准备“上传体验版”前
- 每次准备“提交审核”前（必跑）
- 每次审核被打回后，修改完成再提审前（必跑）
- 出现这些现象时：
  - 体验版能打开但功能不稳定
  - 审核说“无法复现/路径不清晰/权限不一致”
  - 你不确定当前手机装的是不是最新体验版

## 输入（固定表单）
- 目标环境：staging / prod
- AppID：
- 本次版本标识（建议写：版本号 + 构建时间）：
- API Base URL（HTTPS）：
- 是否启用实时：是/否（WSS + /ws）<mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- 是否涉及录音/音频：是/否
- 审核账号/测试账号（如需要登录）：是否准备好（是/否）
- 审核复现路径（一句话）：例如“打开小程序→点‘开始对话’→允许录音→说一句话→看到返回”

## 前置约束（硬规则）
- 必须真机测试（至少 2 台手机更稳）
- 任一关键用例 FAIL：停止提审（先修复/降级/补材料）
- 提审描述必须与实际行为一致（尤其权限/数据收集）

## 冒烟清单（按顺序执行）
### Case 1：确认你跑的是“最新体验版”
- 通过标准：
  - 微信上能看到当前版本号/更新时间（或你在页面里能看到版本标识）
  - 开发者工具显示“上传成功的版本”与你手机一致
- 证据：手机端版本标识截图（或开发者工具截图）

FAIL 的前三定位路径：
1) 手机没有更新到最新体验版（退出重进/重新扫码体验）
2) 你上传的是另一个分支/另一个环境的包
3) 本地构建没更新（打包缓存/配置未生效）

### Case 2：首页可打开 + 无致命错误
- 通过标准：打开首页不卡死、不白屏
- 证据：真机截图

FAIL 的前三定位路径：
1) 包体积/分包/资源加载问题
2) API Base 不可用导致初始化失败
3) 权限弹窗/隐私弹窗处理异常

### Case 3：轻量 API 请求成功（验证白名单+HTTPS）
- 通过标准：请求一条最轻量接口成功（200 或业务成功码）
- 证据：真机调试日志截图/返回结果

FAIL 的前三定位路径：
1) 微信后台 request 合法域名是否配置
2) HTTPS 证书/域名问题
3) 目标环境是否错（staging/prod 混用）

### Case 4：核心链路 3 条（你们必须选出来）
> 这里你们自己定义 3 条最关键动作，例如：
> - 登录/进入主页面
> - 发起一次对话/请求
> - 展示一次结果/历史

- 通过标准：3 条都成功
- 证据：每条成功截图/日志（简要即可）

FAIL 的前三定位路径：
1) 后端业务接口异常（看服务端日志）
2) 小程序端配置漂移（请求到错域名）
3) 权限/隐私流程阻断了用户路径

### Case 5：实时可用（若启用）
- 通过标准：WSS 连接建立成功，完成一次收发 <mccoremem id="03fl75psdph7oiq63zur4f5vg" /></mccoremem>
- 证据：小程序日志或后端 connect 日志关键字

FAIL 的前三定位路径：
1) 微信后台 socket 合法域名是否配置
2) WSS 证书与协议是否正确
3) /ws 路径与网关 Upgrade 是否正确

### Case 6：录音闭环（若涉及录音）
- 通过标准：
  - 允许授权：能开始录音→停止→有结果
  - 拒绝授权：不崩溃，有提示与引导（去设置开启）
- 证据：允许与拒绝两条路径截图

FAIL 的前三定位路径：
1) 权限声明/隐私弹窗不合规或缺失
2) 机型差异（换 2 台验证）
3) 音频上行格式与后端不兼容（base64+format）<mccoremem id="01KHFQYTH1828XWQA5AAM4K3YS" /></mccoremem>

### Case 7：审核材料与“复现路径”准备齐
- 必备项（最少）：
  - 功能描述：一句话讲清用途
  - 隐私政策链接：可访问
  - 数据收集说明：收集什么/为什么/怎么用（各一句）
  - 审核复现路径：审核员按你写的能走通
  - 若需要登录：提供审核账号/免登录路径（否则容易被拒）
- 证据：提审信息草稿截图或文字清单

FAIL 的前三定位路径：
1) 审核无法复现（路径写得不清楚）
2) 权限与描述不一致（说不录音但实际录音）
3) 隐私政策链接打不开或内容缺失

## 上传/提审步骤（小白版流程，固定顺序）
> 这是“你要点哪些按钮”的流程，不涉及代码修改。

1) 微信开发者工具：选择正确 AppID 项目
2) 确认环境配置（API Base / WSS）是目标环境
3) 构建/预览：真机扫预览码跑一遍 Case 1~7
4) 上传：生成体验版
5) 手机端进入体验版：再跑一遍 Case 1~7（重点：Case 1 版本一致）
6) 提交审核：粘贴“复现路径 + 账号信息 + 权限/隐私说明”
7) 审核通过 → 发布

## 输出格式（固定，直接复制到发布记录）
- MP Submit & Release Smoke：PASS / FAIL
- 环境：staging/prod
- 版本标识：____
- Case1 最新体验版：✅/❌（证据：___）
- Case2 首页可打开：✅/❌（证据：___）
- Case3 轻量API：✅/❌（证据：___）
- Case4 核心链路3条：✅/❌（证据：___）
- Case5 实时可用：✅/❌（证据：___）
- Case6 录音闭环：✅/❌（证据：___）
- Case7 审核材料与路径：✅/❌（证据：___）
- 若 FAIL：本次停止提审；前三定位路径：P1___ / P2___ / P3___

## 完成标准（本 skill 完成的判定）
- PASS：Case1~Case7 全绿（不适用项需标注 N/A）
- 你能清楚说明“审核员怎么走到核心功能”（复现路径明确）
- 提审信息与实际行为一致（尤其权限/数据收集）
