# Skill 5：小程序权限与隐私合规验收（Permission & Privacy Gate）

## 目标
把“小程序能不能通过审核、用户能不能顺利授权使用”这件事一次性验收清楚。  
尤其你们涉及录音/音频，这属于审核重点，最容易被打回。

本 skill 负责确认：
- 需要的权限是否声明清楚（录音等）
- 隐私政策是否可用
- “收集什么、为什么收集、怎么用”是否与实际一致
- 用户拒绝授权时产品是否不崩溃、有提示
- 提审描述与实际功能一致（避免审核拒绝）

## 何时调用（触发条件）
- 每次提审之前（必跑）
- 每次新增/修改权限、录音能力、数据上报、第三方 SDK 之后
- 出现这些现象时：
  - 审核被拒，理由涉及“隐私/权限/收集信息”
  - 用户反馈“录音用不了/一直弹窗/点拒绝就卡死”

## 输入（固定表单）
- 目标环境：staging / prod
- 小程序名称：
- AppID：
- 是否涉及录音：是/否
- 是否上传用户数据到服务器：是/否（录音/文本/图片等）
- 隐私政策链接（必须是可访问的 HTTPS 页面）：`https://____`
- 需要的权限清单（勾选）：录音 / 相机 / 相册 / 位置 / 蓝牙 / 无
- 提审功能描述（一段话）：
- 数据收集说明（一段话，写清楚收集什么与用途）：

## 前置约束（硬规则）
- 隐私政策链接不可访问：直接 FAIL
- 实际用到录音但未声明/未说明用途：直接 FAIL
- 用户拒绝授权会导致功能崩溃：直接 FAIL
- 提审描述与实际不一致：直接 FAIL

## 合规验收清单（逐条勾选）
### Check 1：隐私政策链接可用
- 通过标准：
  - 链接可在手机浏览器正常打开
  - 内容至少包含：收集信息类型、用途、保存/删除、联系方式（格式不强求，但要有）
- 证据：手机打开截图

FAIL 的前三定位路径：
1) 链接是否是 http（必须 https）
2) 是否需要备案/证书问题导致打不开
3) 链接是否写错/跳转到不可达页面

### Check 2：权限声明与实际一致（重点：录音）
- 通过标准：
  - 实际会录音 → 权限清单里必须包含“录音”，且功能描述里说明用途
  - 不录音 → 不要申请录音权限（避免审核疑虑）
- 证据：提审描述片段截图/文字

FAIL 的前三定位路径：
1) 代码里是否调用了录音 API，但文案没写
2) 文案写了“不会收集”，但实际上传到服务器
3) 权限申请时机是否太早（用户一打开就要权限容易被拒）

### Check 3：用户授权流程（允许/拒绝/再次授权）
- 通过标准：
  - 第一次触发录音时才弹授权（不要一进来就弹）
  - 用户点“拒绝”后：页面不崩、有明确提示“需要录音权限才能使用此功能”
  - 提供再次授权的引导（例如“去设置开启”）
- 证据：允许与拒绝两种路径各一张截图（或录屏）

FAIL 的前三定位路径：
1) 是否缺少“拒绝授权”分支处理
2) 是否没有清晰提示导致用户以为是 bug
3) 是否需要在设置页引导用户开启权限

### Check 4：数据收集说明（你们最容易踩的审核点）
- 通过标准：用大白话说明这三件事（每条 1 句就够）：
  - 收集什么：例如“用户录音音频/识别后的文本”
  - 为什么收集：例如“用于语音识别与生成回答”
  - 怎么处理：例如“仅用于本次对话处理/不用于广告（按你们真实情况写）”
- 证据：收集说明文本

FAIL 的前三定位路径：
1) 是否写得过于空泛（审核看不懂）
2) 是否与实际行为矛盾（实际上传但写不上传）
3) 是否遗漏敏感信息类型（录音/位置信息等）

### Check 5：第三方 SDK/接口（如有）
- 通过标准：
  - 若使用第三方语音/统计 SDK：在隐私政策中提及（至少说明“第三方服务”）
  - 不要收集与业务无关的信息（例如没用到位置就别申请位置）
- 证据：依赖清单/提及说明（简写即可）

FAIL 的前三定位路径：
1) 是否引入了统计/推送等 SDK 未告知
2) 是否权限申请过多（审核敏感）
3) 是否存在“暗采集”风险点

### Check 6：审核口径一致（类目/功能描述/截图）
- 通过标准：
  - 类目与功能匹配
  - 功能描述与实际一致（你写“语音输入”就必须能跑通）
  - 提供的截图/演示路径能让审核员一眼走通核心功能
- 证据：提审信息截图（类目 + 功能描述 + 截图/路径说明）

FAIL 的前三定位路径：
1) 类目不匹配（常见拒审原因）
2) 审核无法复现功能（路径不清晰/需要登录但无测试账号）
3) 体验版本身不稳定（先跑 Skill 6 冒烟）

## 验收点（可量化）
- A1：隐私政策链接可访问（HTTPS，手机可打开）
- A2：权限申请最小化（只申请需要的权限）
- A3：允许/拒绝授权两条路径都不崩溃且有提示
- A4：收集说明三句话完整且与实际一致
- A5：提审描述与实际功能一致，审核路径清晰

## 输出格式（固定，直接复制到发布记录）
- Permission & Privacy Gate：PASS / FAIL
- AppID：____
- 隐私政策链接可用：✅/❌（证据：___）
- 权限声明一致：✅/❌（证据：___）
- 授权流程（允许/拒绝/再次授权）：✅/❌（证据：___）
- 数据收集说明：✅/❌（证据：___）
- 第三方 SDK 提及：✅/❌（证据：___）
- 审核口径一致（类目/描述/截图/路径）：✅/❌（证据：___）
- 若 FAIL：本次停止；前三定位路径：P1___ / P2___ / P3___

## 完成标准（本 skill 完成的判定）
- 你能给出 PASS/FAIL
- 每项都有证据（最简单就是截图+几句文字）
- FAIL 时停止提审，先修正文案/配置/授权兜底，再重跑本 skill
